Bridge from M00–M03

In M01 you saw that H puts a qubit in equal superposition exploring 0 and 1 simultaneously. In M03 you saw that entanglement correlates qubits across the whole system. Grover's algorithm combines both: it starts with equal superposition across all possible answers, then repeatedly amplifies the probability of the correct answer while suppressing the wrong ones. By the time you measure, the right answer is highly probable not certain, but highly likely.

Grover's Algorithm

The first quantum algorithm with a proven speedup over classical computers. Searches an unsorted database of N items in O(√N) steps instead of O(N). Real, verified, limited by noise on current hardware.

01 The problem searching without structure

Imagine you have a phonebook with one million entries, but it is completely unsorted names are in random order. You want to find one specific name. Classically, you have no choice but to check entries one by one. On average you will check half the entries before finding it half a million checks. In the worst case, you check all one million.

This is called unstructured search. There is no pattern to exploit, no shortcut, no clever indexing that helps. The classical lower bound is O(N) you must check at least some linear fraction of the entries. No classical algorithm can do better.

Grover's algorithm does better. It finds the target entry in approximately √N steps. For one million entries: about 1,000 steps instead of 500,000. This is a quadratic speedup not exponential, but proven, unconditional, and applies to any unstructured search problem.

02 The quantum strategy three steps

Grover's algorithm works by repeatedly applying two operations: the oracle and the diffusion operator. Together they form one Grover iteration. Each iteration increases the probability of measuring the target state.

1
Initialise equal superposition
Apply H to all qubits. Every possible answer now has equal amplitude 1/√N. The system is simultaneously "considering" all N possibilities.
2
Oracle mark the target
Apply the oracle gate. It flips the phase of the target state multiplying its amplitude by −1. The amplitude magnitudes are unchanged but the target now has negative phase. This "marks" it without revealing it.
3
Diffusion amplify the marked state
Apply the diffusion operator (also called inversion about the mean). This reflects all amplitudes about their average value. Because the target has negative amplitude, it gets amplified above the average. All others get suppressed. Repeat steps 2–3 approximately √N/4 times, then measure.
🧠 Analogy noise-cancelling headphones for wrong answers

Imagine all N candidates at equal volume. The oracle plays the target candidate's sound out of phase it does not cancel, it just marks it differently. The diffusion operator then uses interference: the out-of-phase target gets louder (constructive interference) while all others get quieter (destructive interference). After √N repetitions, the target is loud enough to hear clearly above the noise.

03 Amplitude amplification the geometric picture

The clearest way to understand why Grover's works is geometrically. Think of the quantum state as a vector in a two-dimensional space. One axis represents the target state. The other axis represents the uniform superposition of all non-target states.

Initially the state vector is very close to the non-target axis because with N items, the target has amplitude 1/√N which is tiny. Each Grover iteration rotates the state vector by a fixed angle toward the target axis. After approximately π/(4·arcsin(1/√N)) ≈ π√N/4 iterations, the vector points almost directly at the target axis. Measuring then gives the target with high probability.

💡 How many iterations?

The optimal number of Grover iterations is approximately π√N/4. For N=4: 1 iteration. For N=8: 2–3 iterations. For N=1024: ~25 iterations. Applying too many iterations rotates past the target and the probability decreases again it is sinusoidal. This is why you must calculate the right number of iterations in advance.

04 The oracle what it is and why it is hard

The oracle is a quantum gate that recognises the target state and flips its phase. In circuit terms: if the input is the target state |x⟩, the oracle outputs −|x⟩. For all other states, it does nothing.

For simple cases like "find the state |101⟩ among all 3-qubit states", building the oracle is straightforward use a multi-controlled Z gate that triggers exactly on |101⟩. For real-world problems, building the oracle is the hard part.

The oracle problem in practice
For Grover's to help with a real problem, you need to encode the problem's "checking" function as a reversible quantum circuit. For example, to search for a password that hashes to a specific value, you would need to implement the hash function as a quantum circuit. This is technically possible but the resulting circuit is very deep which means it is vulnerable to NISQ noise. This is one of the main reasons Grover's speedup is difficult to realise on current hardware.
Grover's circuit: H gates → (Oracle + Diffusion) repeated k=⌊π√N/4⌋ times → Measure

05 The proven O(√N) speedup

The speedup is exactly quadratic. If classical search needs T steps, Grover's needs √T steps. This is provably optimal no quantum algorithm can do unstructured search faster.

⚠ Common overstatement
✗  "Grover's gives exponential speedup over classical search"
✓  Grover's gives quadratic speedup. Classical: O(N). Grover's: O(√N). For N=2²⁵⁶: classical needs 2²⁵⁵ steps, Grover's needs 2¹²⁸. Still astronomically large for AES-256.
⚠ What this means for cryptography

Grover's halves the effective key length of symmetric encryption. AES-256 effectively becomes AES-128 against a quantum attacker. NIST has already recommended doubling key lengths as a precaution. But AES-128 is still considered secure 2¹²⁸ operations is still computationally infeasible even with a quantum computer. RSA is broken by Shor's algorithm (M05) not Grover's.

06 NISQ reality where Grover's breaks today

On current NISQ hardware, Grover's algorithm only demonstrates a speedup for very small N. Here is why:

The depth problem
Each Grover iteration requires the oracle circuit plus the diffusion circuit. For a 3-qubit (N=8) Grover's search, the circuit depth is manageable. But for N=1024 (10 qubits), each iteration is much deeper, and you need ~25 iterations. The total circuit depth quickly exceeds NISQ coherence limits noise washes out the answer before amplification completes.
Practical NISQ limit
On IBM hardware today, Grover's reliably demonstrates speedup for N ≤ 16 (4 qubits). Beyond that, noise degrades the amplification and the classical computer wins again. The theoretical speedup over classical search only materialises on fault-tolerant hardware which requires error correction not yet available.

07 When Grover's actually matters today

Despite the NISQ limitations, Grover's matters for three reasons right now:

1. It proves quantum advantage is possible
Grover's is the clearest proof that quantum computers can outperform classical computers on a well-defined task. This motivates the entire field. It is not hype it is a theorem.
2. Amplitude amplification is used everywhere
The technique Grover's uses amplitude amplification is a subroutine in dozens of other quantum algorithms. Understanding Grover's gives you the intuition for how quantum speedups work more broadly.
3. Future hardware will unlock the speedup
When fault-tolerant quantum computers arrive (estimated 10–20 years), Grover's speedup will apply to real cryptographic and optimisation problems. Building intuition now means being ready when the hardware catches up.

Define the grover_n8 function, which builds a 3-qubit Grover circuit targeting |101⟩. It starts with H on all three qubits for equal superposition, then loops n_iterations times each iteration applies the oracle (X on q1, then CCZ, then X on q1 to flip phase only on |101⟩) followed by the diffusion operator (H → X → CCZ → X → H on all qubits).

nb02_grover_n8.pystep 1 define grover_n8 function (init + oracle + diffusion loop)
from qiskit import QuantumCircuit, transpile from qiskit_aer import AerSimulator import matplotlib.pyplot as plt import numpy as np def grover_n8(n_iterations): """Grover's for N=8, target=|101⟩, variable iterations.""" qc = QuantumCircuit(3, 3) # Initialise: equal superposition qc.h([0, 1, 2]) for _ in range(n_iterations): qc.barrier() # Oracle: mark target |101⟩ (q2=1, q1=0, q0=1) # CCZ flips phase when q0=1, q1=1, q2=1 # X on q1 first so q1=0 → q1=1 for the CCZ qc.x(1) qc.ccz(0, 1, 2) # Toffoli gate qc.x(1) qc.barrier() # Diffusion operator: H → X → CCZ → X → H qc.h([0, 1, 2]) qc.x([0, 1, 2]) qc.ccz(0, 1, 2) qc.x([0, 1, 2]) qc.h([0, 1, 2]) qc.measure([0, 1, 2], [0, 1, 2]) sim = AerSimulator() counts = sim.run( transpile(qc, sim), shots=2000 ).result().get_counts() return counts.get('101', 0) / 2000 # probability of target

from qiskit import QuantumCircuit, transpile from qiskit_aer import AerSimulator from qiskit.circuit.library import GroverOperator from qiskit.algorithms import AmplificationProblem import numpy as np import matplotlib.pyplot as plt # Use Qiskit's built-in Grover for N=16 (4 qubits), target=|1010⟩ n_qubits = 4 # N = 2^4 = 16 target = '1010' sim = AerSimulator() def run_grover(k): """Run Grover's with k iterations, return probability of target.""" oracle = QuantumCircuit(n_qubits) # Mark |1010⟩: X on q1,q3 (0-indexed) so CCZ fires on |1111⟩ oracle.x([1, 3]) oracle.h(n_qubits-1) oracle.mcx(list(range(n_qubits-1)), n_qubits-1) oracle.h(n_qubits-1) oracle.x([1, 3]) qc = QuantumCircuit(n_qubits, n_qubits) qc.h(range(n_qubits)) # uniform superposition go = GroverOperator(oracle) for _ in range(k): qc.compose(go, inplace=True) qc.measure(range(n_qubits), range(n_qubits)) counts = sim.run( transpile(qc, sim), shots=2000 ).result().get_counts() return counts.get(target, 0) / 2000 max_iter = 10 iters = list(range(1, max_iter+1)) probs = [run_grover(k) for k in iters] # Theoretical prediction: sin²((2k+1)·arcsin(1/√N)) theta = np.arcsin(1/np.sqrt(16)) theory = [np.sin((2*k+1)*theta)**2 for k in iters] optimal_k = int(np.round(np.pi/(4*theta)-0.5)) print(f"N=16: optimal iterations = {optimal_k} (theory: π√N/4 ≈ {np.pi*4/4:.1f})") fig, ax = plt.subplots(figsize=(8, 4)) ax.plot(iters, probs, 'o-', color='#1A4FDB', label='Simulated', linewidth=2) ax.plot(iters, theory, '--', color='#C0392B', label='Theory sin²', linewidth=1.5, alpha=0.7) ax.axvline(optimal_k, color='#1A7A4A', linestyle=':', label=f'Optimal k={optimal_k}') ax.axhline(0.0625, color='gray', linestyle='--', alpha=0.3, label='Uniform (6.25%)') ax.set_xlabel('Number of Grover iterations') ax.set_ylabel('P(target)') ax.set_title("Grover's N=16 sinusoidal amplitude amplification") ax.set_ylim(0, 1.1) ax.legend() plt.tight_layout(); plt.savefig('nb03_amplitude_plot.png'); plt.show()

from qiskit import QuantumCircuit, transpile from qiskit_aer import AerSimulator from qiskit_aer.noise import NoiseModel, depolarizing_error import matplotlib.pyplot as plt # Simple 2-qubit Grover for N=4 on ideal vs noisy def grover_n4_circuit(): qc = QuantumCircuit(2, 2) qc.h([0, 1]) qc.cz(0, 1) # oracle qc.h([0,1]); qc.x([0,1]) qc.cz(0, 1) # diffusion qc.x([0,1]); qc.h([0,1]) qc.measure([0,1], [0,1]) return qc qc = grover_n4_circuit() states = ['00', '01', '10', '11'] # Ideal ideal_counts = AerSimulator().run( transpile(qc, AerSimulator()), shots=2000 ).result().get_counts() # Noisy (1% error similar to IBM today) nm = NoiseModel() nm.add_all_qubit_quantum_error(depolarizing_error(0.01, 1), ['h', 'x']) nm.add_all_qubit_quantum_error(depolarizing_error(0.02, 2), ['cz']) noisy_counts = AerSimulator(noise_model=nm).run( transpile(qc, AerSimulator(noise_model=nm)), shots=2000 ).result().get_counts() ideal_probs = [ideal_counts.get(s,0)/2000 for s in states] noisy_probs = [noisy_counts.get(s,0)/2000 for s in states] print("Ideal: target |11⟩ =", f"{ideal_probs[3]:.1%}") print("Noisy: target |11⟩ =", f"{noisy_probs[3]:.1%}") print("Uniform would be: 25.0%") x = range(4) fig, ax = plt.subplots(figsize=(7, 4)) ax.bar([i-0.2 for i in x], ideal_probs, 0.35, label='Ideal', color='#1A4FDB', alpha=0.8) ax.bar([i+0.2 for i in x], noisy_probs, 0.35, label='Noisy (1% error)', color='#C0392B', alpha=0.7) ax.axhline(0.25, color='gray', linestyle='--', alpha=0.4, label='Uniform (25%)') ax.set_xticks(list(x)); ax.set_xticklabels(states) ax.set_ylabel('Probability'); ax.legend() ax.set_title("Grover's N=4 ideal vs noisy (1% gate error)") plt.tight_layout(); plt.savefig('nb04_noisy_grover.png'); plt.show()
⚠ What you will observe

Even at 1% gate error (similar to IBM hardware today), the target state probability drops from ~100% (ideal) to ~80–85%. The non-target states pick up probability they should not have. At higher error rates the distribution flattens toward uniform the quantum advantage disappears completely. This is the core NISQ challenge for Grover's.

import numpy as np import matplotlib.pyplot as plt N_values = [4, 8, 16, 64, 256, 1024, 4096, 16384] # Classical: average N/2 checks classical = [N/2 for N in N_values] # Grover's: π√N/4 iterations grover = [np.pi * np.sqrt(N) / 4 for N in N_values] # Speedup factor speedup = [c/g for c, g in zip(classical, grover)] print(f"{'N':>8} {'Classical':>12} {'Grover':>10} {'Speedup':>10}") print("-" * 48) for N, c, g, s in zip(N_values, classical, grover, speedup): print(f"{N:>8} {c:>12.0f} {g:>10.1f} {s:>9.1f}×") # Plot log scale fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 4)) # Left: steps comparison ax1.loglog(N_values, classical, 'o-', color='#C0392B', label='Classical O(N)', linewidth=2) ax1.loglog(N_values, grover, 'o-', color='#1A4FDB', label="Grover's O(√N)", linewidth=2) ax1.set_xlabel('Database size N (log scale)') ax1.set_ylabel('Steps required (log scale)') ax1.set_title('Classical vs Grover steps to find target') ax1.legend(); ax1.grid(alpha=0.3) # Right: speedup factor ax2.semilogx(N_values, speedup, 'o-', color='#1A7A4A', linewidth=2) ax2.set_xlabel('Database size N (log scale)') ax2.set_ylabel('Speedup factor (Classical / Grover)') ax2.set_title("Grover speedup grows as √N") ax2.grid(alpha=0.3) plt.suptitle("Grover's quadratic speedup over classical search", fontweight='bold') plt.tight_layout() plt.savefig('nb05_comparison.png') plt.show()

The speedup grows as √N doubling N increases the speedup by √2. At N=16,384 the speedup is 81×. At N=1,000,000 it would be ~637×. This is the proven quadratic advantage of Grover's algorithm over classical unstructured search.

QAOA – Quantum Approximate Optimization Algorithm

How quantum circuits tackle NP-hard combinatorial problems. From MaxCut to routing to scheduling the QUBO formulation, the cost Hamiltonian, the QAOA circuit, and when it actually helps over classical solvers.

01 The problem class QAOA is built for

QAOA targets combinatorial optimisation problems problems where you must choose the best assignment from a finite but astronomically large set of possibilities. Every choice is binary: include or exclude, go left or go right, assign or don't assign.

The canonical example is MaxCut: given a graph of nodes connected by weighted edges, partition the nodes into two groups such that the total weight of edges crossing between the groups is maximised. This sounds abstract, but MaxCut is the structure underlying circuit design (VLSI placement), network partitioning, and financial portfolio selection.

Why these problems are hard exponential search spaces
MaxCut on 20 nodes: 220 = 1,048,576 possible partitions. Exhaustive search is feasible.

MaxCut on 50 nodes: 250 ≈ 1015 partitions. At 109 checks/second, exhaustive search takes 11 days.

MaxCut on 100 nodes: 2100 ≈ 1030 partitions. The age of the universe is ~4 × 1017 seconds. Classical exhaustive search is completely infeasible.

This is why these problems are classified as NP-hard no polynomial-time classical algorithm is known. Both classical heuristics (simulated annealing, branch-and-bound) and quantum algorithms (QAOA, D-Wave annealing) find good-but-not-guaranteed-optimal solutions.

The key word in QAOA is Approximate. QAOA does not find the provably optimal solution it finds a good solution, with a quality guarantee that improves as the number of circuit layers p increases. This is an honest tradeoff: more layers = better approximation = more circuit depth = more noise. On current hardware p = 1 or p = 2 is practical.

✓ Connection to your advisory platform

When you typed "shortest delivery route for 20 stops" into the advisory platform, it recommended QAOA. Now you know why: VRP is a combinatorial optimisation problem. Each routing decision is binary "does vehicle visit city A before city B?" and the total number of possible routes is factorial in the number of stops. QAOA is designed for exactly this structure. In this module you will implement QAOA for MaxCut and then adapt it to a small routing problem.

02 QUBO the language quantum optimisation speaks

Before a quantum computer can tackle an optimisation problem, the problem must be written in a language it understands. That language is QUBO Quadratic Unconstrained Binary Optimisation.

A QUBO problem has the form: minimise xTQx where x is a vector of binary variables (each 0 or 1) and Q is a matrix encoding the costs and interactions between variables. Every combinatorial optimisation problem can be expressed in this form including MaxCut, routing, scheduling, and portfolio selection.

MaxCut QUBO in three steps
Step 1 Variables: Assign one binary variable xi per node. xi = 0 means node i is in group 0; xi = 1 means group 1.

Step 2 Objective: For each edge (i,j) with weight wij, it crosses the cut when xi ≠ xj. This contributes wij(xi + xj − 2xixj) to the cut value. Summing over all edges gives the total cut value to maximise.

Step 3 Minimise the negative: QUBO minimises, so negate: minimise −Σ wij(xi + xj − 2xixj). This is now in the form xTQx with Qij = 2wij for connected nodes and Qii = −Σj wij for the diagonal.

The QUBO matrix Q is the complete description of the problem. Every off-diagonal entry Qij is an interaction between variables i and j in quantum terms, this becomes a ZZ coupling term between qubits i and j. Every diagonal entry Qii becomes a local field a Z rotation on qubit i.

💡 QUBO to Ising the natural quantum form

Quantum hardware speaks Ising model, not binary variables. The transformation is: substitute xi = (1 − zi)/2 where zi ∈ {−1,+1} is the Ising spin variable. A binary variable x=0 corresponds to spin z=+1 (qubit |0⟩), and x=1 corresponds to z=−1 (qubit |1⟩). Every QUBO problem becomes an Ising problem in this substitution, with the Hamiltonian H = Σij Jij ZiZj + Σi hi Zi. This is called the cost Hamiltonian the quantum object QAOA minimises.

03 The QAOA circuit two alternating layers

QAOA is a variational algorithm. Like the VQC in M06/M07, it has trainable parameters that a classical optimiser adjusts. The circuit structure is fixed by the problem the parameters are found by optimisation.

A QAOA circuit with p layers has exactly 2p parameters: p angles γ (gamma) for the cost layers and p angles β (beta) for the mixer layers. For a problem with n binary variables, the circuit uses n qubits.

1
Initial state equal superposition
Apply Hadamard H to every qubit. This puts all qubits into |+⟩ = (|0⟩ + |1⟩)/√2, creating an equal superposition over all 2n possible assignments. The quantum computer is simultaneously "considering" every possible solution before the optimisation begins.
2
Cost layer encode the objective
Apply exp(−iγ Hc) where Hc is the cost Hamiltonian. For MaxCut, each edge (i,j) contributes a ZZ gate: RZZ(2γwij) on qubits i and j. Each node contributes an RZ gate. This layer adds phase to each basis state proportional to how good that solution is good solutions accumulate different phase than bad ones. γ controls how much.
3
Mixer layer allow exploration
Apply exp(−iβ Hm) where Hm = −Σi Xi is the transverse-field mixer. In practice: RX(2β) on every qubit. This layer mixes the phases, allowing the quantum state to tunnel between different solutions. Without it, the circuit would just be phase kickback with no optimisation. β controls how much mixing happens.
4
Repeat p times and measure
Alternate cost and mixer layers p times, with separate γk and βk for each layer k. After the p-th layer, measure all qubits. The measurement samples a candidate solution. The expected cost ⟨Hc⟩ is estimated by averaging many shots. The classical optimiser (COBYLA or SPSA) then adjusts γ and β to reduce ⟨Hc⟩.
QAOA circuit p=1 on a 4-node MaxCut problem
QAOA p=1 · 4 qubits · H → Rz+RZZ cost layer (γ) → Rx mixer (β) → Measure

04 The hybrid loop classical optimiser drives the quantum circuit

QAOA is a hybrid quantum-classical algorithm. The quantum circuit evaluates how good a set of parameters is. The classical computer finds better parameters. This loop runs until convergence.

1
Initialise γ, β randomly
Start with random angles γk ∈ [0, 2π] and βk ∈ [0, π]. For p layers, initialise 2p parameters.
2
Run circuit, estimate ⟨Hc⟩
Execute the QAOA circuit with current γ, β. Run 1000–4000 shots. The fraction of shots in each bitstring estimates the probability of each solution. The expected cost ⟨Hc⟩ = Σ P(x) × cost(x) is the objective to minimise.
3
Classical optimiser updates parameters
COBYLA (gradient-free, good for noisy circuits) or SPSA (stochastic gradient, fewer circuit evaluations) adjusts γ and β to reduce ⟨Hc⟩. Unlike VQC, the parameter-shift rule is not used here gradient-free optimisers are preferred because circuit noise makes gradient estimates unreliable.
4
Extract solution
After convergence, run the circuit once more and take the most frequently sampled bitstring as the solution. This is the QAOA's "answer" to the optimisation problem a good but not necessarily optimal assignment.
⚠ Barren plateaus in QAOA

Just like VQCs, QAOA suffers from barren plateaus as p increases. For large p on many-qubit problems, gradients of ⟨Hc⟩ vanish exponentially the optimiser has no signal. In practice this limits useful QAOA to small p (1–3) on current hardware. Good parameter initialisation (using the INTERP or Fourier strategies) helps but does not fully solve the problem.

05 QAOA approximation ratio what you actually get

For MaxCut specifically, QAOA with p=1 achieves an approximation ratio of at least 0.6924 meaning the cut value it finds is at least 69.24% of the optimal cut value. This is a provable guarantee for any graph.

⚠ Classical solvers are still better for most sizes

The Goemans-Williamson classical algorithm achieves a 0.878 approximation ratio better than QAOA at p=1 or p=2. OR-Tools and Gurobi find exact solutions for hundreds of nodes in seconds. QAOA does not outperform the best classical solvers on any practically relevant problem size today. The algorithm is important as a building block for future fault-tolerant quantum advantage, not as a production solver right now.

06 Real applications routing, scheduling, portfolio

MaxCut is the textbook QAOA problem, but the same QUBO formulation applies to a wide range of real problems. Each maps differently to binary variables and cost terms.

ProblemBinary variableQubits neededCost terms
MaxCut (n nodes)xi = which partition node i is innZZij for each edge
VRP/TSP (n cities)xit = "is city i visited at time t?"n² (one-hot) or n·log₂n (compact)Distance ZZ + constraint penalties
Job scheduling (J jobs, T slots)xjt = "is job j in slot t?"J × TPrecedence + resource constraint penalties
Portfolio (n assets)xi = "include asset i?"nReturn/covariance ZZ terms
💡 Why routing needs n² qubits

To specify a full TSP tour, you need to know "which city is visited at each position in the route." For n cities and n positions, that is n² binary decisions: xit = 1 means city i is at position t in the tour. With n=20 cities you need 400 binary variables 400 qubits in the standard encoding. The "constraint penalties" ensure the solution is a valid tour (each city visited exactly once). These penalties dominate the qubit count and are why compact encodings (reducing to n·log₂n) are an active research area.

07 Honest picture where QAOA stands today

ClaimReality
"QAOA outperforms classical solvers"Not demonstrated on any practically relevant problem size. Classical solvers (OR-Tools, Gurobi, Goemans-Williamson) remain superior for all sizes QAOA can currently run.

"QAOA is the right algorithm to learn for combinatorial optimisation"Yes it is the canonical gate-model quantum optimisation algorithm, widely implemented on IBM hardware, and the foundation for understanding all VQE-based optimisation work.

Brute-force the optimal cut

With only 4 nodes there are 2⁴ = 16 possible partitions. Enumerate them all to find the optimal cut value. This gives us the ground truth for evaluating QAOA.

nb00_setup.pybrute force
def cut_value(x, G): """Compute cut value for binary assignment x (0/1 per node).""" total = 0.0 for u, v, d in G.edges(data=True): if x[u] != x[v]: # edge crosses the cut total += d['weight'] return total n = G.number_of_nodes() best_cut, best_x = 0, None for i in range(2**n): x = [(i >> k) & 1 for k in range(n)] cv = cut_value(x, G) if cv > best_cut: best_cut, best_x = cv, x[:] print(f"Optimal partition : {best_x}") print(f"Optimal cut value : {best_cut}") print(f"Bitstring : {''.join(map(str, best_x))}") print(f"\nGroup 0: nodes {[i for i,v in enumerate(best_x) if v==0]}") print(f"Group 1: nodes {[i for i,v in enumerate(best_x) if v==1]}")

1
Build the QAOA circuit

We build the circuit layer by layer. The cost layer applies RZZ gates for each edge and RZ gates for each node. The mixer layer applies RX to every qubit. Parameters γ and β are Qiskit Parameter objects they will be assigned values by the optimiser.

nb01_qaoa_p1.pybuild circuit
from qiskit import QuantumCircuit from qiskit.circuit import Parameter import numpy as np n_qubits = G.number_of_nodes() # 4 gamma = Parameter('γ') beta = Parameter('β') def build_qaoa_p1(G, gamma, beta): """QAOA circuit with p=1 for MaxCut on graph G.""" qc = QuantumCircuit(n_qubits) # ── Initial state: equal superposition ── qc.h(range(n_qubits)) qc.barrier() # ── Cost layer: encode the MaxCut Hamiltonian ── # For each edge (u,v,w): apply RZZ(2*gamma*w) on qubits u,v # RZZ(θ) = exp(-i θ/2 · ZZ) = CNOT · Rz(θ) · CNOT for u, v, d in G.edges(data=True): w = d['weight'] qc.rzz(2 * gamma * w, u, v) # Diagonal cost terms (single-node fields from QUBO) for node in G.nodes(): h_i = -sum(d['weight'] for _, _, d in G.edges(node, data=True)) / 2 if abs(h_i) > 1e-9: qc.rz(2 * gamma * h_i, node) qc.barrier() # ── Mixer layer: transverse field ── # exp(-i β Σ Xᵢ) = Rx(2β) on each qubit for q in range(n_qubits): qc.rx(2 * beta, q) qc.measure_all() return qc qc_qaoa = build_qaoa_p1(G, gamma, beta) print(qc_qaoa.draw(output='text', fold=80))

Define the cost function and run hybrid optimisation

The cost function runs the circuit, samples measurements, and computes ⟨Hc⟩. COBYLA minimises this it is gradient-free and robust to the noise in circuit output.

nb01_qaoa_p1.pyoptimize
from qiskit_aer import AerSimulator from qiskit.compiler import transpile from scipy.optimize import minimize simulator = AerSimulator() SHOTS = 2000 cost_history = [] def compute_cost(params): """Estimate ⟨Hc⟩ by running circuit and computing expected cut value.""" g_val, b_val = params # Bind parameters and transpile bound_qc = qc_qaoa.assign_parameters({gamma: g_val, beta: b_val}) t_qc = transpile(bound_qc, simulator) result = simulator.run(t_qc, shots=SHOTS).result() counts = result.get_counts() # Compute expected cut value from measurement frequencies expected_cut = 0.0 for bitstring, count in counts.items(): # Qiskit orders bits right-to-left: bitstring[0] = q_{n-1} x = [int(bitstring[-1-i]) for i in range(n_qubits)] expected_cut += (count / SHOTS) * cut_value(x, G) cost_history.append(-expected_cut) # negate for minimisation return -expected_cut # scipy.minimize minimises, so negate # Optimise starting from random initial angles np.random.seed(42) x0 = np.random.uniform(0, 2*np.pi, 2) result = minimize(compute_cost, x0, method='COBYLA', options={'maxiter': 200, 'rhobeg': 0.5}) print(f"Optimal γ = {result.x[0]:.4f}, β = {result.x[1]:.4f}") print(f"Expected cut value: {-result.fun:.4f}") print(f"Optimal cut (brute force): {best_cut:.4f}") print(f"Approximation ratio: {-result.fun/best_cut:.4f}")

def build_qaoa(G, p): """General QAOA circuit for p layers. Returns (circuit, gammas, betas).""" n = G.number_of_nodes() gammas = [Parameter(f'γ{k}') for k in range(p)] betas = [Parameter(f'β{k}') for k in range(p)] qc = QuantumCircuit(n) qc.h(range(n)) for k in range(p): qc.barrier() # Cost layer k for u, v, d in G.edges(data=True): qc.rzz(2*gammas[k]*d['weight'], u, v) for node in G.nodes(): h_i = -sum(d['weight'] for _, _, d in G.edges(node, data=True)) / 2 if abs(h_i) > 1e-9: qc.rz(2*gammas[k]*h_i, node) # Mixer layer k qc.barrier() for q in range(n): qc.rx(2*betas[k], q) qc.measure_all() return qc, gammas, betas # Build a harder 6-node graph for comparison G6 = nx.random_regular_graph(3, 6, seed=42) for u, v in G6.edges(): G6[u][v]['weight'] = 1.0 # unweighted for simplicity # Brute force optimal best_6 = max(cut_value([(i>k)&1 for k in range(6)], G6) for i in range(64)) print(f"6-node optimal cut: {best_6}") # Compare p=1 and p=2 for p in [1, 2]: qc_p, gs, bs = build_qaoa(G6, p) params_sym = gs + bs def cost_p(vals): bnd = qc_p.assign_parameters(dict(zip(params_sym, vals))) t = transpile(bnd, simulator) cnt = simulator.run(t, shots=1000).result().get_counts() return -sum(cut_value([int(b[-1-i]) for i in range(6)], G6)*c/1000 for b, c in cnt.items()) x0 = np.random.uniform(0, 2*np.pi, 2*p) res = minimize(cost_p, x0, method='COBYLA', options={'maxiter': 300, 'rhobeg': 0.5}) ratio = -res.fun / best_6 print(f"p={p}: cut={-res.fun:.3f}, optimal={best_6}, ratio={ratio:.4f}, depth={qc_p.depth()}")

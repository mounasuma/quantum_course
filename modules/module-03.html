Entanglement & Bell States

Entanglement is the phenomenon that Einstein called "spooky action at a distance" and it is the most powerful resource in quantum computing. By the end of this module you will have created entanglement, measured it, and understood exactly what it does and does not mean.

01 What entanglement is NOT clearing the myths first

Entanglement is the most misunderstood concept in quantum computing. Before explaining what it is, let's clear up the three things it is not because the myths are everywhere, including in mainstream science writing.

🚫 Myth 1
✗  "Entanglement allows faster-than-light communication"
✓  Measuring one entangled qubit gives a random result. You cannot control that result. Therefore you cannot send any information you just get correlated randomness. No signal travels anywhere.
🚫 Myth 2
✗  "Entangled qubits are physically connected by some invisible link"
✓  There is no physical connection. Entanglement is a property of the shared quantum state a mathematical correlation encoded in the joint wave function of the two qubits.
🚫 Myth 3
✗  "Entanglement means the qubits always have the same value"
✓  Before measurement, neither qubit has a definite value at all. After measurement, they are correlated but depending on the Bell state, they might be the same or opposite. The correlation is the key, not equality.

02 What entanglement actually is

Two qubits are entangled when their quantum states cannot be described independently of each other. Formally: the two-qubit state cannot be written as a product of two single-qubit states.

Here is the clearest way to understand it. Consider two separate qubits, each in superposition. Their combined state has four possibilities: 00, 01, 10, 11 each with some probability. If you measure one qubit and find 0, the other qubit might still be 0 or 1 with some probability the two measurements are independent.

Now create an entangled pair. Measuring one qubit instantly determines the other regardless of distance. Not because a signal travels between them. But because they share a single quantum state that spans both qubits simultaneously. The outcome is random, but the outcomes are correlated.

Bell's theorem (1964) proved experimentally that entangled qubits genuinely have no definite value until measured. This is not a matter of ignorance it is physical reality.

03 The four Bell states

There are exactly four maximally entangled two-qubit states, called the Bell states or Bell basis. They are the most entangled states two qubits can be in.

NameStateWhat it meansCorrelation
|Φ+⟩ (|00⟩ + |11⟩) / √2 Always same: both 0 or both 1 Correlated
|Φ−⟩ (|00⟩ − |11⟩) / √2 Always same but with phase difference Correlated
|Ψ+⟩ (|01⟩ + |10⟩) / √2 Always opposite: one 0 and one 1 Anti-correlated
|Ψ−⟩ (|01⟩ − |10⟩) / √2 Always opposite with phase difference Anti-correlated

The most important for now is |Φ+⟩ this is what the H + CNOT circuit creates, and it is what you will measure most often. The minus signs in |Φ−⟩ and |Ψ−⟩ represent phase differences that are invisible in measurement statistics alone but matter when combined with other gates.

04 How to create entanglement the H + CNOT circuit

You already saw this circuit in Module 02. Now let's understand exactly why it creates entanglement, step by step.

Step 1: H puts q₀ in superposition. Step 2: CNOT entangles q₁ to q₀. Result: Bell state |Φ+⟩.
Step by step: why H + CNOT creates entanglement
Start: Both qubits in |0⟩. Combined state: |00⟩.

After H on q₀: q₀ is in superposition |+⟩ = (|0⟩ + |1⟩)/√2. q₁ is still |0⟩. Combined state: (|0⟩ + |1⟩)/√2 ⊗ |0⟩ = (|00⟩ + |10⟩)/√2. These two qubits are still separable not yet entangled.

After CNOT (q₀ controls q₁): When q₀ = |0⟩, q₁ stays |0⟩ → contributes |00⟩. When q₀ = |1⟩, q₁ flips to |1⟩ → contributes |11⟩. Final state: (|00⟩ + |11⟩)/√2 = |Φ+⟩.

This state cannot be factored. You cannot write it as (something about q₀) ⊗ (something about q₁). They are entangled.

05 What happens when you measure a Bell state

When you measure both qubits of a |Φ+⟩ Bell state, you get either 00 or 11, each with 50% probability. You never get 01 or 10. This perfect correlation across any distance, instantaneously is the signature of entanglement.

Measurement outcomes for |Φ+⟩ = (|00⟩ + |11⟩)/√2
q₀ = 0 q₁ = 0 ~50% probability ✓
q₀ = 0 q₁ = 1 0% never happens ✗
q₀ = 1 q₁ = 0 0% never happens ✗
q₀ = 1 q₁ = 1 ~50% probability ✓

The 01 and 10 outcomes have exactly zero probability not approximately zero, not rarely, but exactly never. This perfect correlation cannot be explained by any classical hidden variable theory. Bell's theorem proves it. This is one of the most experimentally verified facts in all of physics.

💡 Why this matters for quantum computing

Entanglement allows quantum computers to process exponentially many states simultaneously. When n qubits are entangled, their joint state cannot be described by n separate qubit states it requires 2ⁿ amplitudes. This is the source of quantum computing's exponential state space. Without entanglement, a quantum computer is not more powerful than a classical one.

06 Where entanglement appears in real algorithms

Entanglement is not just a curiosity it is the computational resource that makes quantum algorithms work. Here is where it appears in the algorithms you will build in this course:

Grover's algorithm (Module 04)
The oracle and diffusion operator create entanglement across all qubits. This is what allows the amplitude of the correct answer to be amplified across the entire superposition simultaneously not qubit by qubit.
QAOA (Module 06)
The cost Hamiltonian layer creates entanglement between qubits representing different variables. This is how the algorithm explores correlations between variables which is exactly what makes combinatorial optimisation hard classically.
Quantum teleportation (this module, hands-on)
A Bell pair is used to transmit a qubit state from one location to another using only classical communication. The entanglement does the work the classical channel carries the correction information.

07 Entanglement and decoherence why it breaks

Entanglement is fragile. Any interaction with the environment a stray photon, a vibration, a tiny electromagnetic fluctuation can collapse the entangled state. This is called decoherence, and it is one of the central engineering challenges of quantum computing.

T₂ the dephasing time
T₂ is the timescale over which quantum phase information (including entanglement) is lost due to environmental noise. On IBM's best superconducting hardware today, T₂ is approximately 100–300 microseconds. Your quantum circuit must complete before T₂ expires or the entanglement degrades into classical noise.
⚠ The practical consequence

On NISQ hardware, you can only maintain entanglement across a handful of gate operations before decoherence destroys it. This is why NISQ algorithms are designed to be shallow not because the algorithms are simple, but because the hardware cannot sustain deep entangled circuits.

Import the necessary libraries and define the bell_circuit helper function. It accepts two boolean flags x0 and x1 that control whether an X gate is applied to each qubit before the standard H+CNOT sequence, letting you generate any of the four Bell states from one function.

nb02_all_bell_states.pystep 1 imports + define bell_circuit function
from qiskit import QuantumCircuit, transpile from qiskit_aer import AerSimulator import matplotlib.pyplot as plt sim = AerSimulator() states = ['00', '01', '10', '11'] def bell_circuit(x0=False, x1=False): """Create Bell state. x0/x1: apply X gate before H+CNOT.""" qc = QuantumCircuit(2, 2) if x0: qc.x(0) # flip q₀ first if x1: qc.x(1) # flip q₁ first qc.h(0) qc.cx(0, 1) qc.measure([0,1], [0,1]) return qc

Define all four Bell state configurations using the x0/x1 flag combinations, then create each circuit and run it 1000 shots to get the measurement probability distribution. |Φ+⟩ and |Φ−⟩ are correlated (only 00 and 11 appear), while |Ψ+⟩ and |Ψ−⟩ are anti-correlated (only 01 and 10 appear).

This is an important lesson: quantum states carry more information than their measurement probabilities alone.

NB 04 · Reality check
Entanglement under noise watching correlations degrade
Run the Bell state circuit on a noisy simulator. Watch how noise introduces 01 and 10 outcomes that should be impossible. The correlation coefficient drops below 1. This is decoherence in action.
⏱ ~20 min⚠️ NISQ reality

Define the bell_correlation function, which builds a Bell state circuit and attaches a depolarizing noise model to it. Single-qubit gates get the base error rate and the two-qubit CX gate gets twice that, since two-qubit gates are always noisier on real hardware.

nb04_noisy_entanglement.pystep 1 define bell_correlation function with noise model
from qiskit import QuantumCircuit, transpile from qiskit_aer import AerSimulator from qiskit_aer.noise import NoiseModel, depolarizing_error import matplotlib.pyplot as plt def bell_correlation(error_rate): """Run Bell state with given error rate. Return correlation coefficient.""" qc = QuantumCircuit(2, 2) qc.h(0); qc.cx(0, 1) qc.measure([0,1], [0,1]) noise_model = NoiseModel() noise_model.add_all_qubit_quantum_error( depolarizing_error(error_rate, 1), ['h'] ) noise_model.add_all_qubit_quantum_error( depolarizing_error(error_rate * 2, 2), ['cx'] ) sim = AerSimulator(noise_model=noise_model) counts = sim.run( transpile(qc, sim), shots=2000 ).result().get_counts() c00 = counts.get('00',0); c11 = counts.get('11',0) c01 = counts.get('01',0); c10 = counts.get('10',0) return (c00+c11-c01-c10) / 2000, counts

Run the Bell state experiment at six different gate error rates from 0% (perfect simulator) up to 10% and print the correlation coefficient alongside a quality label. This table shows how quickly entanglement degrades as hardware imperfections increase.

nb04_noisy_entanglement.pystep 2 test at 6 different error rates
# Test at different noise levels error_rates = [0.0, 0.005, 0.01, 0.02, 0.05, 0.1] correlations = [] print("Error rate → Correlation coefficient → Entanglement quality") for rate in error_rates: corr, _ = bell_correlation(rate) correlations.append(corr) quality = "Perfect" if corr > 0.99 else "Good" if corr > 0.95 else "Degraded" if corr > 0.8 else "Broken" print(f" {rate*100:.1f}% → {corr:.3f} → {quality}")

Plot the correlation coefficient against the gate error rate, adding a dashed green line at 0.95 to mark the usable threshold and a dashed red line at 0.0 for classical random. The downward slope of the curve directly visualises decoherence as noise rises, the quantum entanglement quality falls toward classical noise.

nb04_noisy_entanglement.pystep 3 plot correlation vs error rate
# Plot correlation vs noise fig, ax = plt.subplots(figsize=(7, 4)) ax.plot([r*100 for r in error_rates], correlations, 'o-', color='#1A4FDB', linewidth=2, markersize=6) ax.axhline(0.95, color='#1A7A4A', linestyle='--', alpha=0.5, label='Usable threshold (0.95)') ax.axhline(0.0, color='#C0392B', linestyle='--', alpha=0.3, label='Classical random (0.0)') ax.set_xlabel('Gate error rate (%)') ax.set_ylabel('Correlation coefficient') ax.set_title('Entanglement quality degrades with noise') ax.legend(); ax.set_ylim(0, 1.05) plt.tight_layout() plt.savefig('nb04_noisy_entanglement.png') plt.show()
⚠ What to look for

At 0% error: correlation ≈ 1.000 perfect entanglement. At 1% error (IBM hardware today): correlation ≈ 0.97 still very good. At 5% error (older hardware): correlation ≈ 0.85 degraded. At 10% error: correlation ≈ 0.7 barely above classical random. This shows exactly why gate fidelity matters so much for any quantum algorithm that relies on entanglement.

Superdense coding is the first practical application of entanglement. The protocol: Alice and Bob share a Bell pair created in advance. Alice wants to send one of four messages 00, 01, 10, or 11 to Bob. By applying a gate to only her qubit and then sending it, Bob can decode the 2-bit message by measuring both qubits. 2 bits of classical information transmitted by physically sending 1 qubit.

The encoding table what Alice does to her qubit
To send 00: do nothing (Identity)
To send 01: apply X gate
To send 10: apply Z gate
To send 11: apply X then Z gate

Bob then applies CNOT and H to both qubits and measures. The measurement outcome tells Bob which 2-bit message Alice sent.
nb05_superdense_coding.pypython nb05_superdense_coding.py
from qiskit import QuantumCircuit, transpile from qiskit_aer import AerSimulator sim = AerSimulator() def superdense_encode(message): """ Superdense coding protocol. message: '00', '01', '10', or '11' Returns measurement result (should match message). """ qc = QuantumCircuit(2, 2) # Step 1: Create Bell pair (done in advance, shared by Alice and Bob) qc.h(0) qc.cx(0, 1) qc.barrier() # visual separator # Step 2: Alice encodes her 2-bit message by operating on qubit 0 only if message == '01': qc.x(0) elif message == '10': qc.z(0) elif message == '11': qc.x(0); qc.z(0) # '00': do nothing qc.barrier() # Step 3: Bob decodes by applying CNOT then H, then measuring qc.cx(0, 1) qc.h(0) qc.measure([0,1], [0,1]) counts = sim.run( transpile(qc, sim), shots=100 ).result().get_counts() return max(counts, key=counts.get) # most common outcome # Test all four messages print("Superdense coding sending 2 bits using 1 qubit:") print(f"{'Message sent':20} {'Message received':20} {'Correct?'}") print("-" * 55) for msg in ['00', '01', '10', '11']: received = superdense_encode(msg) status = '✓' if received == msg else '✗' print(f" Sent: {msg:18} Received: {received:18} {status}")
✓ Expected output

All four messages should be received correctly 00→00, 01→01, 10→10, 11→11. This works because the entangled Bell pair acts as a shared resource. Alice manipulates her qubit to encode the message into the joint entangled state; Bob's decoding circuit reads out both bits. The entanglement is doing the information-carrying work.

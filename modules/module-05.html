Introduction to Optimisation

What combinatorial optimisation is, why it gets hard so fast, what classical tools exist today, and how to formulate problems as QUBO the format quantum optimisers actually accept.

01 What is an optimisation problem?

An optimisation problem has three parts: variables (the decisions you can make), constraints (the rules those decisions must follow), and an objective function (the thing you want to minimise or maximise).

Simple example: you are packing a bag for a trip. The variables are which items to pack. The constraint is the bag's weight limit. The objective is to maximise the total value of what you bring. This is the knapsack problem one of the most studied optimisation problems in computer science.

🧠 Analogy decision making under constraints

Every real engineering decision is an optimisation problem. A delivery company wants the shortest route visiting 50 locations. A hospital wants the most efficient schedule for 200 patients across 30 doctors. A fund manager wants the portfolio with highest return for a given level of risk. In each case: many choices, constraints, one objective to optimise.

The reason these problems are interesting and hard is that the number of possible solutions grows explosively with problem size. For small problems, you can check every solution and pick the best. For large problems, that is impossible. Finding smarter approaches is what optimisation is about.

02 Why optimisation gets hard combinatorial explosion

You saw combinatorial explosion in M00 with the city routing problem. Let's make it concrete with a simpler example: the maximum cut problem, or MaxCut.

Given a graph a set of nodes connected by edges divide the nodes into two groups such that the number of edges between the two groups is maximised. This sounds simple. For a graph with 5 nodes, there are 2⁵ = 32 possible ways to partition the nodes. Easily checked by hand.

For 50 nodes: 2⁵⁰ ≈ 10¹⁵ possible partitions. For 100 nodes: 2¹⁰⁰ ≈ 10³⁰. The number of solutions grows exponentially. Checking them all becomes impossible long before real-world problem sizes.

📌 Why MaxCut matters
MaxCut appears in circuit design, statistical physics, financial portfolio optimisation, and network clustering. It is also the canonical benchmark problem for QAOA every QAOA paper uses MaxCut. Understanding MaxCut well means you will understand what QAOA is actually optimising in M06.

03 NP-hard what it means and why it matters

Problems like MaxCut, TSP (Travelling Salesman), and knapsack belong to a complexity class called NP-hard. This means no known algorithm can solve them exactly in polynomial time the time required grows faster than any polynomial as the problem size grows.

What NP-hard means in plain English
Verifying a solution is easy given a partition of nodes, you can quickly count how many edges are cut. But finding the best solution requires checking an exponentially large space of possibilities. There is no clever shortcut that avoids this. This is a mathematical result, not a hardware limitation.
⚠ Common confusion
✗  "NP-hard means impossible to solve"
✓  NP-hard means impossible to solve exactly for large instances in reasonable time. In practice, we use heuristics that find near-optimal solutions very quickly and that is usually good enough.

This is the key insight for quantum optimisation: quantum computers are not targeting the easy problems they are targeting this specific class of hard combinatorial problems where classical exact solvers fail at scale. The question is whether quantum heuristics can outperform classical heuristics. The honest answer today is: not yet, but potentially yes for specific problem structures.

05 Simulated annealing the classical heuristic to understand

Simulated annealing is the classical algorithm most directly comparable to quantum annealing (M07) and gives intuition for how QAOA works. Understanding it well makes the quantum versions much clearer.

1
Start with a random solution
Pick any valid assignment of variables. This is your current solution. Calculate its cost (objective function value).
2
Make a small random change
Flip one variable, swap two cities, move one node to the other group. Calculate the new cost.
3
Accept or reject the change
If the new solution is better always accept. If worse accept with probability e^(−ΔE/T), where T is the "temperature". High temperature = accept bad moves freely. Low temperature = only accept improvements.
4
Gradually lower the temperature
As T → 0, the algorithm becomes greedy. Early on it explores widely (escaping local minima). Later it exploits (converging to a good solution).
🧠 Analogy cooling metal

When metal cools slowly from liquid, atoms settle into a low-energy crystalline structure the global minimum. Cool too fast (quenching) and atoms freeze into a disordered, high-energy structure a local minimum. Simulated annealing mimics slow cooling: random exploration early, greedy exploitation late. Quantum annealing does the same thing but exploits quantum tunnelling to escape local minima more efficiently.

06 QUBO the format quantum optimisers use

Before you can run a problem on a quantum optimiser whether QAOA or D-Wave you must express it as a QUBO: Quadratic Unconstrained Binary Optimisation problem.

QUBO has three requirements. First: all variables must be binary either 0 or 1. Second: the objective function must be quadratic at most two variables multiplied together. Third: unconstrained constraints must be absorbed into the objective as penalty terms.

QUBO general form
minimise: xᵀQx = Σᵢ Qᵢᵢxᵢ + Σᵢ＜ⱼ Qᵢⱼxᵢxⱼ
x = binary vector (each xᵢ ∈ {0, 1})
Q = upper-triangular matrix of coefficients
Diagonal Qᵢᵢ = linear terms (cost of setting variable i to 1)
Off-diagonal Qᵢⱼ = quadratic terms (interaction between variables i and j)

The key insight is that every binary optimisation problem can be expressed as a QUBO it just requires careful reformulation. MaxCut, TSP, knapsack, graph colouring, portfolio optimisation all can be written as xᵀQx. This is why QUBO is the universal input format for quantum optimisers.

MaxCut as QUBO concrete example
For MaxCut on a graph with nodes {1,2,3} and edges {(1,2), (2,3), (1,3)}:

Variable xᵢ = 1 if node i is in group A, 0 if in group B.

An edge (i,j) is cut if and only if xᵢ ≠ xⱼ, which equals xᵢ + xⱼ − 2xᵢxⱼ.

Maximise: Σ(i,j)∈E (xᵢ + xⱼ − 2xᵢxⱼ)
Equivalently minimise: Σ(i,j)∈E (2xᵢxⱼ − xᵢ − xⱼ)

This is exactly the QUBO form. The Q matrix has −1 on diagonal entries for each node's edge count, and +2 on off-diagonal entries for each edge.

nb00_maxcut_bruteforce.pystep 1 build graph
import networkx as nx import itertools import time import matplotlib.pyplot as plt G = nx.Graph() G.add_nodes_from([0, 1, 2, 3, 4, 5]) G.add_edges_from([ (0,1), (0,2), (1,3), (2,3), (2,4), (3,5), (4,5), (1,4) ]) print(f"Graph: {G.number_of_nodes()} nodes, {G.number_of_edges()} edges") print(f"Brute force checks needed: 2^{G.number_of_nodes()} = {2**G.number_of_nodes()}") def count_cut_edges(G, partition): """Count edges between group A (partition=1) and group B (partition=0).""" return sum(1 for u, v in G.edges() if partition[u] != partition[v])

Try every possible way of splitting the 6 nodes into two groups all 64 combinations. For each partition compute the cut value and keep track of the best one found.

nb00_maxcut_bruteforce.pystep 2 brute force
best_cut = 0 best_part = None n = G.number_of_nodes() start = time.perf_counter() for bits in itertools.product([0, 1], repeat=n): partition = dict(enumerate(bits)) cut = count_cut_edges(G, partition) if cut > best_cut: best_cut = cut best_part = partition elapsed = (time.perf_counter() - start) * 1000 print(f"\nOptimal MaxCut: {best_cut} edges cut") print(f"Partition: {best_part}") print(f"Time: {elapsed:.2f} ms")

Import the libraries, build the same 16-node graph as before, and define a cut_value helper that counts edges crossing between the two groups.

nb01_simulated_annealing.pystep 1 setup
import networkx as nx import random import math import matplotlib.pyplot as plt G = nx.random_regular_graph(3, 16, seed=42) n = G.number_of_nodes() def cut_value(G, assignment): return sum(1 for u,v in G.edges() if assignment[u] != assignment[v])

Define the simulated annealing function. It starts from a random assignment, then repeatedly flips one random node. If the flip improves the cut it always accepts; if it worsens, it accepts with probability e^(ΔE/T) a probability that shrinks as the temperature T cools down.

nb01_simulated_annealing.pystep 2 SA function
def simulated_annealing(G, T_start=10.0, T_end=0.01, steps=5000): n = G.number_of_nodes() curr = {i: random.randint(0,1) for i in range(n)} best = curr.copy() best_cut = cut_value(G, curr) curr_cut = best_cut history = [curr_cut] alpha = (T_end / T_start) ** (1 / steps) T = T_start for _ in range(steps): node = random.randint(0, n-1) new_curr = curr.copy() new_curr[node] = 1 - new_curr[node] new_cut = cut_value(G, new_curr) delta = new_cut - curr_cut if delta > 0 or random.random() < math.exp(delta / T): curr = new_curr curr_cut = new_cut if curr_cut > best_cut: best = curr.copy() best_cut = curr_cut history.append(curr_cut) T *= alpha return best, best_cut, history

Run SA three times to show it produces consistent results despite being stochastic. Then run it once more at 8000 steps and plot the cut value over iterations you'll see wild fluctuations early when temperature is high, then gradual settling as it cools.

nb01_simulated_annealing.pystep 3 run and plot
print(f"MaxCut on {n}-node graph:") for run in range(3): _, best_cut, _ = simulated_annealing(G) print(f" Run {run+1}: MaxCut = {best_cut}") best_assign, best_cut, history = simulated_annealing(G, steps=8000) print(f"\nFinal best: {best_cut} edges cut") rolling_max = [max(history[:i+1]) for i in range(len(history))] fig, ax = plt.subplots(figsize=(9, 4)) fig.patch.set_facecolor('#1E1E2E') ax.set_facecolor('#181825') ax.plot(history, color='#89B4FA', linewidth=1, alpha=0.7, label='Cut value') ax.plot(rolling_max, color='#A6E3A1', linewidth=2, label='Best found so far') ax.axhline(best_cut, color='#F38BA8', linestyle='--', linewidth=1.5, label=f'Final best: {best_cut}') ax.set_xlabel('Iteration', color='#CDD6F4') ax.set_ylabel('Cut value', color='#CDD6F4') ax.set_title('Simulated annealing convergence MaxCut 16 nodes', color='#CDD6F4') ax.tick_params(colors='#CDD6F4') ax.legend(labelcolor='#CDD6F4', facecolor='#313244') for spine in ax.spines.values(): spine.set_edgecolor('#45475A') plt.tight_layout() plt.savefig('nb01_sa_convergence.png', facecolor='#1E1E2E') plt.show()
The key lines
delta = new_cut - curr_cutHow much better (positive) or worse (negative) the proposed change is.
math.exp(delta / T)Acceptance probability for worse moves. When T is high, this is close to 1 accept almost anything. When T is low, this is close to 0 only accept improvements.
T *= alphaExponential cooling. Temperature drops by factor alpha each step. This is the "schedule" how fast you cool determines solution quality.
What each part does full walkthrough
curr = {i: random.randint(0,1)}Start with a completely random assignment each node randomly in group 0 or group 1. SA does not need a good starting point; it will explore from here.
alpha = (T_end/T_start)**(1/steps)Cooling rate. Each step the temperature is multiplied by alpha (a number slightly below 1). This gives exponential decay from T_start down to T_end over the number of steps.
node = random.randint(0, n-1)Pick a random node to flip. This is the "move" a small perturbation to the current solution. One node changes from group 0 to group 1 or vice versa.
delta = new_cut - curr_cutHow much the cut value changes. Positive = improvement (new solution cuts more edges). Negative = worsening (fewer cut edges). Zero = no change.
math.exp(delta / T)Acceptance probability for worse moves (delta < 0). When T is large: exp(small negative) ≈ 1, accept almost anything. When T is tiny: exp(large negative) ≈ 0, reject bad moves. This is the core of SA.
T *= alphaCool down by multiplying by alpha. After all steps, T has decayed from T_start to T_end. The algorithm transitions from exploration (hot) to exploitation (cold).
rolling_max = [max(history[:i+1])]Track the best cut value found so far at each step. This "best found so far" line on the plot shows convergence when it plateaus, SA has found its best solution.
✓ What to expect

The cut value will fluctuate wildly early (high temperature accepting bad moves). Then gradually stabilise as temperature drops. The green "best found so far" line should plateau near the optimal. Compare with brute force from NB 00 SA should find the same or very close result in a fraction of the time.

from ortools.constraint_solver import routing_enums_pb2 from ortools.constraint_solver import pywrapcp import numpy as np import matplotlib.pyplot as plt import time # Generate random city coordinates np.random.seed(42) n_cities = 15 coords = np.random.rand(n_cities, 2) * 100 # Build distance matrix def build_distance_matrix(coords): n = len(coords) dist = np.zeros((n, n), dtype=int) for i in range(n): for j in range(n): dist[i][j] = int(np.linalg.norm(coords[i] - coords[j]) * 100) return dist dist_matrix = build_distance_matrix(coords) # Solve TSP with OR-Tools manager = pywrapcp.RoutingIndexManager(n_cities, 1, 0) routing = pywrapcp.RoutingModel(manager) def distance_callback(from_idx, to_idx): return dist_matrix[manager.IndexToNode(from_idx)][manager.IndexToNode(to_idx)] transit_cb = routing.RegisterTransitCallback(distance_callback) routing.SetArcCostEvaluatorOfAllVehicles(transit_cb) search_params = pywrapcp.DefaultRoutingSearchParameters() search_params.first_solution_strategy = ( routing_enums_pb2.FirstSolutionStrategy.PATH_CHEAPEST_ARC ) start = time.perf_counter() solution = routing.SolveWithParameters(search_params) elapsed = (time.perf_counter() - start) * 1000 # Extract route route = [] idx = routing.Start(0) while not routing.IsEnd(idx): route.append(manager.IndexToNode(idx)) idx = solution.Value(routing.NextVar(idx)) route.append(route[0]) # return to start total_dist = solution.ObjectiveValue() / 100 print(f"OR-Tools TSP ({n_cities} cities):") print(f" Route: {' → '.join(map(str, route))}") print(f" Total distance: {total_dist:.1f} units") print(f" Time: {elapsed:.2f} ms") print(f"\n⚠ QAOA cannot solve {n_cities}-city TSP competitively on NISQ hardware.") print(f" Classical wins today. This is your benchmark.") # Visualise route fig, ax = plt.subplots(figsize=(7, 6)) fig.patch.set_facecolor('#1E1E2E') ax.set_facecolor('#181825') for i in range(len(route)-1): ax.annotate("", xy=coords[route[i+1]], xytext=coords[route[i]], arrowprops=dict(arrowstyle="->", color="#89B4FA", lw=1.8)) ax.scatter(coords[:,0], coords[:,1], color='#F38BA8', s=80, zorder=5) for i, (x,y) in enumerate(coords): ax.annotate(str(i), (x,y), textcoords="offset points", xytext=(6,4), color='#CDD6F4', fontsize=9) ax.set_title(f'OR-Tools TSP {n_cities} cities, distance={total_dist:.1f}, time={elapsed:.1f}ms', color='#CDD6F4') ax.tick_params(colors='#CDD6F4') for spine in ax.spines.values(): spine.set_edgecolor('#45475A') plt.tight_layout() plt.savefig('nb02_ortools_tsp.png', facecolor='#1E1E2E') plt.show()

The MaxCut objective is: maximise the number of cut edges. In QUBO we need to minimise, so we negate: minimise −(cut edges). For each edge (i,j), the cut contribution is xᵢ + xⱼ − 2xᵢxⱼ. Summing over all edges and negating gives us the QUBO objective.

nb03_qubo.pypython nb03_qubo.py
import networkx as nx import numpy as np import itertools # Simple 4-node graph for clarity G = nx.Graph() G.add_nodes_from([0, 1, 2, 3]) G.add_edges_from([(0,1), (1,2), (2,3), (0,3)]) n = G.number_of_nodes() print("Graph edges:", list(G.edges())) print("Nodes:", list(G.nodes())) # ── Build QUBO matrix Q ── # MaxCut: minimise -(cut) = minimise Σ(i,j)∈E (2xᵢxⱼ - xᵢ - xⱼ) # Q[i][i] = linear coefficient of xᵢ (sum of -1 for each edge containing i) # Q[i][j] = quadratic coefficient of xᵢxⱼ = +2 for each edge (i,j) Q = np.zeros((n, n)) for i, j in G.edges(): Q[i][i] -= 1 # linear: -1 per edge for node i Q[j][j] -= 1 # linear: -1 per edge for node j Q[i][j] += 2 # quadratic: +2 per edge (i,j) print("\nQUBO matrix Q:") print(Q) # ── Verify by evaluating all solutions ── # QUBO energy = x @ Q @ x (treating x as column vector) print("\nAll solutions QUBO energy and cut value:") print(f"{'x':>12} {'QUBO energy':>12} {'Cut edges':>10}") print("-" * 40) best_energy = float('inf') best_solution = None for bits in itertools.product([0,1], repeat=n): x = np.array(bits) energy = x @ Q @ x # QUBO evaluation cut = sum(1 for u,v in G.edges() if x[u] != x[v]) print(f"{str(bits):>12} {energy:>12.1f} {cut:>10}") if energy < best_energy: best_energy = energy best_solution = bits print(f"\nOptimal QUBO solution: x = {best_solution}") print(f"Minimum energy: {best_energy} = maximum cut") print("\n✓ The minimum QUBO energy corresponds to the maximum cut.") print(" This Q matrix is exactly what you will pass to QAOA in M06.")
The QUBO construction every step explained
G.add_edges_from([...])Build a 4-node cycle graph. Small enough to check all 2⁴=16 solutions by hand, large enough to see the QUBO structure clearly.
Q = np.zeros((n, n))Initialise the QUBO matrix as all zeros. We will fill in the coefficients edge by edge. Upper-triangular convention: Q[i][j] only when i ≤ j.
Q[i][i] -= 1Linear term. For each edge (i,j), subtract 1 from both diagonal entries Q[i][i] and Q[j][j]. These represent the −xᵢ and −xⱼ terms in the MaxCut objective. A node with 3 edges gets Q[i][i] = −3.
Q[i][j] += 2Quadratic interaction term. For each edge (i,j), add 2 to the off-diagonal entry. This represents the +2xᵢxⱼ term. When both nodes are in the same group (xᵢ=xⱼ=1), this +2 cancels the two −1 diagonals edge not cut.
x @ Q @ xEvaluate QUBO energy for assignment x. This is xᵀQx matrix multiplication. Lower energy = better solution. The minimum energy assignment is the maximum cut partition.
energy = x @ Q @ x vs cut = sum(...)Cross-check: verify that −energy equals the cut value. If QUBO is formulated correctly, minimising energy maximises cut edges. You will see this in the output table.
best_solution = bitsTrack the assignment with the lowest QUBO energy. This is the answer a quantum annealer (D-Wave) or QAOA circuit will try to find. In M06 you will pass this same Q matrix to QAOA and compare.

What machine learning is, how it works classically, where it struggles, and how quantum circuits can act as trainable models. From loss functions to variational quantum classifiers the full conceptual arc.

01 What machine learning actually is

Machine learning is optimisation. You have a model with parameters. You have data. You have a loss function that measures how wrong the model is. You adjust the parameters to reduce the loss. That is it.

Every ML algorithm linear regression, random forests, deep neural networks follows this pattern. The differences are in the model architecture, the loss function, and the optimisation algorithm. The core idea is always: find parameter values that minimise loss on training data, and generalise to unseen data.

The three ingredients of every ML model
Model: A parameterised function f(x; θ) that maps input x to prediction. For a neural network, θ are the weights. For a VQC, θ are the rotation angles.

Loss function: A number measuring how wrong f(x; θ) is on labelled data. Cross-entropy for classification. MSE for regression. Lower loss = better model.

Optimiser: An algorithm that updates θ to reduce loss. Gradient descent follows the slope of the loss landscape downhill. Adam, COBYLA, SPSA are all variants of this idea.

The training loop runs these three ingredients repeatedly: forward pass (compute predictions), backward pass (compute gradients), update (adjust θ). After thousands of iterations, θ settles at values that produce good predictions. This loop is identical in classical ML and quantum ML only the model changes.

03 Where classical ML struggles the motivation for QML

Classical ML is extremely good at learning from data. So what is the motivation for quantum ML at all? There are three genuine arguments and one that is mostly hype.

Argument 1 Exponential feature spaces (genuine)
A quantum state on n qubits lives in a 2ⁿ-dimensional Hilbert space. Classically, computing in a 2ⁿ-dimensional space requires exponential memory. A quantum circuit accesses this space natively. For certain kernel-based methods, this exponential feature space might allow QML to find patterns that classical methods cannot at least in theory.
Argument 2 Natural fit for quantum data (genuine)
If your input data is already quantum e.g. output of a quantum chemistry simulation, or measurements from a quantum sensor a quantum classifier processes it without the encoding bottleneck. This is the most credible near-term use case for QML.
Argument 3 Quantum speedup for linear algebra (future)
Some quantum algorithms for linear systems (HHL algorithm) offer exponential speedup. If this could be integrated into ML training, it might speed up certain operations. But dequantisation results (classical algorithms with similar scaling) have weakened this argument significantly.
⚠ The hype argument
✗  "Quantum computers process all possibilities simultaneously so QML is exponentially faster"
✓  Processing in superposition does not automatically give a speedup. You still need to measure, which collapses the state. The quantum advantage must come from carefully designed interference not just superposition. Most QML claims have not demonstrated a proven speedup over classical ML.

05 Encoding classical data into quantum states

Before you can classify data with a quantum circuit, you need to encode the data into quantum states. This is called the feature map or data encoding. It is one of the most important design decisions in QML and one where there is no settled answer.

1
Angle encoding used in M07
Map each feature to a rotation angle. Feature xᵢ becomes Ry(xᵢ) on qubit i. Simple, requires n qubits for n features. Your VQC uses this: 784 pixel values → PCA → 4 features → 4 rotation angles on 4 qubits.
2
Amplitude encoding
Encode N features into 2ⁿ amplitudes of n qubits. Exponentially more compact 2ⁿ features in n qubits. But state preparation is expensive and often cancels the advantage.
3
IQP / ZZ feature map
Apply data-dependent entangling gates between qubits. Creates a quantum kernel in an exponentially large feature space. Used in quantum SVM. Theoretically motivated but practically unproven.
💡 The encoding bottleneck

Loading n classical bits into a quantum state generically takes O(2ⁿ) gates exponential cost. This is called the input problem or data loading problem. It means any quantum speedup from processing in exponential state space may be cancelled by the cost of loading the data. This is one of the most serious open problems in QML.

06 Variational Quantum Circuits the VQC architecture

A Variational Quantum Circuit (VQC) is a parameterised quantum circuit used as a machine learning model. It has three parts: an encoding layer that loads the data, a variational layer with trainable parameters, and a measurement layer that reads out the prediction.

VQC architecture the three layers
Encoding layer: Ry(xᵢ) on each qubit i. Maps classical features into quantum rotation angles. This layer has no trainable parameters it is fixed by the data.

Variational layer (repeated L times): Ry(θᵢ) and Rz(φᵢ) on each qubit (trainable), followed by CNOT entangling gates between neighbouring qubits. The parameters θ, φ are learned during training. More layers = more expressibility = more parameters to optimise.

Measurement layer: Measure ⟨Z⟩ on qubit 0 (binary) or ⟨Zᵢ⟩ on all qubits (multiclass). The expectation values are fed into a classical classifier or directly thresholded for the final prediction.
VQC circuit structure your M07 circuit
4-qubit VQC · 2 layers · Angle encoding → Ry+Rz+CNOT ring → ⟨Z⟩ measurement

Your M07 circuit uses exactly this structure: 4 qubits (one per PCA feature), 2 variational layers each with Ry+Rz on every qubit and a ring of CNOT gates, and measurement of ⟨Z⟩ for binary classification or all four ⟨Zᵢ⟩ for 10-class. Total trainable parameters: 2 layers × 4 qubits × 2 angles = 16 parameters.

07 The VQC training loop how it learns

Training a VQC follows the same loop as training a classical neural network. The key difference is where the gradient comes from.

1
Forward pass run the circuit
Encode a training sample x into the circuit. Run the circuit with current parameters θ. Measure ⟨Z⟩. This is the prediction a number between −1 and +1. For binary classification: ⟨Z⟩ ≥ 0 → class 0, ⟨Z⟩ < 0 → class 1.
2
Compute loss
For binary: MSE between prediction and target (+1 for class 0, −1 for class 1). For 10-class: cross-entropy between softmax of [⟨Z₀⟩,...,⟨Z₃⟩]×linear_layer and one-hot label. Lower loss = better fit to training data.
3
Compute gradient the parameter shift rule
Backpropagation does not work directly on quantum circuits. Instead use the parameter shift rule: ∂⟨Z⟩/∂θᵢ = [⟨Z⟩(θᵢ+π/2) − ⟨Z⟩(θᵢ−π/2)] / 2. Each gradient requires two extra circuit evaluations per parameter. For 16 parameters: 32 extra circuit runs per training step.
4
Update parameters
Adam optimiser adjusts θ in the direction that reduces loss. On a simulator this is fast. On real hardware each circuit evaluation takes real QPU time which is why training on hardware is impractical and training on simulator then deploying is the standard approach.
💡 Why training on simulator and testing on QPU

Training a VQC with 16 parameters for 40 epochs requires thousands of circuit evaluations. On IBM QPU with a 5-15 minute queue per job, training would take days. Instead: train completely on simulator (fast, noiseless). Save the trained weights. Deploy to QPU only for inference one forward pass per test sample. This is exactly what your M07 code does.

08 Honest picture where QML stands today

Quantum machine learning is one of the most overhyped areas in all of technology. Before doing the M07 project, you should have a clear-eyed view of what the field has and has not achieved.

ClaimReality
"QML outperforms classical ML"Not demonstrated on any real-world problem. Classical deep learning wins on all standard benchmarks by large margins.
"VQCs have exponential expressibility"Theoretically true but practically limited by barren plateaus gradients vanish exponentially as circuit depth grows, making training impossible.
"QML is useful for quantum data"Most credible near-term claim. Classifying quantum states or quantum chemistry outputs avoids the data loading bottleneck.
"30% on 10-class Fashion-MNIST is impressive"Yes 3× better than random (10%), matching published results, with only 16 parameters and 4 qubits. It is a proof of concept that the approach works, not a production classifier.
⚠ The barren plateau problem

As VQC depth and qubit count grow, gradients of the cost function vanish exponentially the landscape becomes flat and the optimiser has no signal. This means VQCs do not scale the way classical neural networks do. Training a 20-qubit VQC is not straightforwardly harder than 4 qubits it may be fundamentally intractable. This is the central open problem in QML.

Despite these limitations, doing the M07 project is genuinely valuable. You will understand the full pipeline from data to QPU. You will see what real hardware noise does to a trained model. You will have hands-on experience that most quantum computing practitioners do not have. And you will be in a much better position to evaluate future QML claims honestly.

import pennylane as qml from pennylane import numpy as pnp import numpy as np # ── Config: exactly as in your M07 binary VQC ── N_QUBITS = 4 N_LAYERS = 2 # PennyLane simulator device noiseless, runs on CPU dev = qml.device("default.qubit", wires=N_QUBITS)

Define the VQC as a decorated Python function an encoding layer that loads the 4 PCA features as Ry rotation angles, then two variational layers of Ry+Rz rotations followed by a ring of CNOT entangling gates, and finally a Pauli-Z measurement on qubit 0 that returns a value in [−1, +1].

nb01_first_vqc.pystep 2 define VQC function
# ── Define the VQC ── @qml.qnode(dev, interface="autograd") def vqc(weights, x): # ── Encoding layer: load data as rotation angles ── for i in range(N_QUBITS): qml.RY(x[i], wires=i) # encode feature xᵢ into qubit i # ── Variational layers: trainable ── for l in range(N_LAYERS): # Single-qubit rotations: Ry and Rz on each qubit for i in range(N_QUBITS): qml.RY(weights[l, i, 0], wires=i) # trainable rotation qml.RZ(weights[l, i, 1], wires=i) # trainable rotation # Entangling layer: ring of CNOTs for i in range(N_QUBITS - 1): qml.CNOT(wires=[i, i + 1]) # nearest-neighbour entanglement qml.CNOT(wires=[N_QUBITS - 1, 0]) # ring closure: last→first # ── Measurement: expectation value of Z on qubit 0 ── return qml.expval(qml.PauliZ(0)) # +1 = T-shirt, -1 = Trouser

Initialise the weight tensor with small random values and run one forward pass on a dummy input this confirms the circuit executes, shows the ⟨Z⟩ output and circuit diagram, and lets you verify the total parameter count is 16 before training begins.

nb01_first_vqc.pystep 3 init weights + forward pass
# ── Initialise random weights ── np.random.seed(42) weights = pnp.array( np.random.uniform(-0.01, 0.01, (N_LAYERS, N_QUBITS, 2)), requires_grad=True ) print(f"Weight shape: {weights.shape}") print(f"Total parameters: {weights.size}") # ── Forward pass on a single input ── x_sample = pnp.array([0.5, 1.2, 0.8, 0.3]) # 4 features after PCA+scaling prediction = vqc(weights, x_sample) print(f"\nForward pass result:") print(f" Input x : {x_sample}") print(f" ⟨Z⟩ output : {float(prediction):.4f}") print(f" Predicted class: {'T-shirt (0)' if float(prediction) >= 0 else 'Trouser (1)'}") # ── Draw the circuit ── print("\nCircuit diagram:") print(qml.draw(vqc)(weights, x_sample)) # ── Count parameters ── print(f"\nParameter count breakdown:") print(f" N_LAYERS × N_QUBITS × 2 = {N_LAYERS} × {N_QUBITS} × 2 = {N_LAYERS*N_QUBITS*2}") print(f" (Ry + Rz per qubit per layer)") print(f" CNOT gates: {N_QUBITS} per layer × {N_LAYERS} layers = {N_QUBITS*N_LAYERS} (no parameters)")
What each part does
qml.device("default.qubit")Create a PennyLane noiseless statevector simulator. "default.qubit" runs on CPU using NumPy. Alternative: "lightning.qubit" for faster C++ simulation. "qiskit.ibmq" to run on IBM hardware.
@qml.qnode(dev, interface="autograd")Decorator that turns the Python function into a quantum node a differentiable quantum circuit. The "autograd" interface allows PennyLane to compute gradients using the parameter shift rule, enabling backpropagation-style training.
qml.RY(x[i], wires=i)Encoding layer. Rotate qubit i by angle x[i]. This loads the i-th PCA feature into the circuit. The angle is from the data not trainable.
qml.RY(weights[l,i,0], wires=i)Variational layer. Rotate qubit i by the trained angle. This parameter will be adjusted by the optimiser during training to minimise the loss.
qml.CNOT(wires=[N_QUBITS-1, 0])Ring closure CNOT connects last qubit back to first qubit. Creates a circular entanglement topology. This ensures all qubits interact with all others across layers, giving the circuit enough expressibility to learn correlations between features.
qml.expval(qml.PauliZ(0))Measure the expectation value of the Z observable on qubit 0. Returns a number in [−1, +1]. ≥0 → T-shirt prediction. <0 → Trouser prediction. This is the output of the forward pass.
requires_grad=TrueTell PennyLane these are trainable parameters. Without this flag, no gradient will be computed and the optimiser cannot update these weights.

nb02_train_vqc.pypython nb02_train_vqc.py
import pennylane as qml from pennylane import numpy as pnp import numpy as np from sklearn.datasets import make_classification from sklearn.preprocessing import MinMaxScaler from sklearn.metrics import accuracy_score import matplotlib.pyplot as plt # ── Toy dataset: 2 features, 2 classes, 60 samples ── np.random.seed(42) X_raw, y = make_classification( n_samples=60, n_features=2, n_redundant=0, n_clusters_per_class=1, random_state=42 ) # Scale to [0, π] required for angle encoding X = MinMaxScaler(feature_range=(0, np.pi)).fit_transform(X_raw) X = pnp.array(X, requires_grad=False) print(f"Dataset: {len(y)} samples, 2 features, 2 classes") # ── 2-qubit VQC (2 features = 2 qubits) ── N_QUBITS = 2 N_LAYERS = 2 dev = qml.device("default.qubit", wires=N_QUBITS) @qml.qnode(dev, interface="autograd") def vqc(weights, x): for i in range(N_QUBITS): qml.RY(x[i], wires=i) for l in range(N_LAYERS): for i in range(N_QUBITS): qml.RY(weights[l, i, 0], wires=i) qml.RZ(weights[l, i, 1], wires=i) qml.CNOT(wires=[0, 1]) return qml.expval(qml.PauliZ(0)) # ── Cost function: MSE between ⟨Z⟩ and target (±1) ── def cost(W, X, y): preds = pnp.array([vqc(W, x) for x in X]) targets = pnp.where(y == 0, 1.0, -1.0) # class 0 → +1, class 1 → -1 return pnp.mean((preds - targets) ** 2) # MSE loss # ── Training loop ── np.random.seed(42) W = pnp.array(np.random.uniform(-0.01, 0.01, (N_LAYERS, N_QUBITS, 2)), requires_grad=True) opt = qml.AdamOptimizer(stepsize=0.02) losses, accs = [], [] print("\nTraining VQC on toy data...") for ep in range(30): # Step: compute gradient and update weights in one call W, loss = opt.step_and_cost(lambda w: cost(w, X, y), W) if (ep + 1) % 5 == 0: preds = np.array([0 if float(vqc(W, x)) >= 0 else 1 for x in X]) acc = accuracy_score(y, preds) losses.append(float(loss)) accs.append(acc) print(f" Epoch {ep+1:3d} | loss={float(loss):.4f} | acc={acc:.3f}") # Plot training curves fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(10, 4)) ax1.plot([5,10,15,20,25,30], losses, 'o-', color='#C0392B', linewidth=2) ax1.set_xlabel('Epoch'); ax1.set_ylabel('MSE Loss') ax1.set_title('VQC Training Loss') ax2.plot([5,10,15,20,25,30], accs, 'o-', color='#1A4FDB', linewidth=2) ax2.axhline(0.5, color='gray', linestyle='--', alpha=0.5, label='Random baseline') ax2.set_xlabel('Epoch'); ax2.set_ylabel('Accuracy') ax2.set_title('VQC Training Accuracy') ax2.set_ylim(0, 1.1); ax2.legend() plt.tight_layout() plt.savefig('nb02_vqc_training.png') plt.show() print(f"\nFinal accuracy: {accs[-1]:.1%}") print("You are ready for M07 Fashion-MNIST on IBM QPU.")
What each part does
make_classification(n_features=2)Generate a toy 2-class dataset with 2 features. Simple and fast perfect for verifying the training loop works before scaling to Fashion-MNIST. With 2 features we can also visualise the decision boundary.
targets = pnp.where(y==0, 1.0, -1.0)Convert integer class labels (0,1) to VQC targets (+1,−1). The VQC outputs ⟨Z⟩ ∈ [−1,+1], so we use signed targets. Class 0 maps to +1 (VQC should output positive). Class 1 maps to −1.
pnp.mean((preds - targets)**2)MSE loss. Measures how far the VQC predictions are from the signed targets. If prediction=+0.8 and target=+1: loss contribution = (0.8−1)² = 0.04. Averaged over the batch.
opt.step_and_cost(...)PennyLane's combined step: compute gradient using parameter shift rule, update W using Adam, return new W and loss value. This is equivalent to optimizer.zero_grad(), loss.backward(), optimizer.step() in PyTorch.
lambda w: cost(w, X, y)Lambda wraps the cost function so step_and_cost knows which argument is the one to differentiate with respect to (W). The lambda freezes X and y as constants.

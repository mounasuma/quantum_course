💡 What you will actually learn from the results

The most valuable part of this project is not the accuracy numbers it is understanding why the numbers are what they are. Why does sim-HW agreement stay high (85-90%) while accuracy drops? Because the circuit structure survived transpilation the noise degraded individual measurements, not the circuit logic. Why did Trouser classify perfectly (100%) but T-shirt failed (0%)? Because the VQC learned a strong Trouser signal in one PCA component, but noise pushed all T-shirt predictions across the decision boundary. These are real NISQ effects that you measured, not textbook theory.

💡 Why saving transformers matters

The original deploy code re-fitted PCA on only the 20 hardware test samples different from the training preprocessing. This is a data leakage / inconsistency bug. The fix: fit scaler and PCA once on training data, save them, load them in the deploy script. That way both training and hardware inference see the same transformation.

Load the full Fashion-MNIST dataset and normalise pixel intensities with MinMaxScaler, fitting only on training data this per-feature normalisation ensures PCA operates on consistent scales before any compression happens.

nb02_preprocess.pystep 1 load + normalise pixels
import numpy as np import tensorflow as tf import pickle import matplotlib.pyplot as plt from sklearn.decomposition import PCA from sklearn.preprocessing import MinMaxScaler N_QUBITS = 4 CLASS_NAMES = ['T-shirt','Trouser','Pullover','Dress','Coat', 'Sandal','Shirt','Sneaker','Bag','Ankle boot'] # ── Load full dataset ── (X_train_all, y_train_all), (X_test_all, y_test_all) = \ tf.keras.datasets.fashion_mnist.load_data() X_train_all = X_train_all.reshape(-1, 784).astype(np.float32) / 255.0 X_test_all = X_test_all.reshape(-1, 784).astype(np.float32) / 255.0 # ── Step 1: Normalise pixel intensities to [0,1] ── # Already done above by dividing by 255 # But we also normalise per-feature to handle PCA scale differences scaler = MinMaxScaler() X_train_norm = scaler.fit_transform(X_train_all) # fit on TRAIN only X_test_norm = scaler.transform(X_test_all) # transform test with train's params print(f"After MinMaxScaler: range [{X_train_norm.min():.2f}, {X_train_norm.max():.2f}]")

Fit PCA with 4 components on the normalised training data and transform both train and test this compresses 784 pixel dimensions to the 4 directions of maximum variance that will become the qubit rotation angles.

nb02_preprocess.pystep 2 fit PCA(4) + transform
# ── Step 2: PCA compression 784 → 4 ── pca = PCA(n_components=N_QUBITS, random_state=42) X_train_pca = pca.fit_transform(X_train_norm) # fit on TRAIN only X_test_pca = pca.transform(X_test_norm) var_explained = pca.explained_variance_ratio_.sum() * 100 print(f"After PCA(4): {var_explained:.1f}% variance explained") print(f" Component variances: {[f'{v:.1%}' for v in pca.explained_variance_ratio_]}")

Scale the 4 PCA features into the range [0, π] this maps each feature to a valid Ry rotation angle so that 0 leaves the qubit at |0⟩ and π flips it to |1⟩, giving the encoding layer its full expressive range.

nb02_preprocess.pystep 3 scale features to [0,π]
# ── Step 3: Scale PCA features to [0, π] for angle encoding ── ang_scaler = MinMaxScaler(feature_range=(0, np.pi)) X_train_ang = ang_scaler.fit_transform(X_train_pca) # fit on TRAIN only X_test_ang = ang_scaler.transform(X_test_pca) print(f"After angle scaling: range [{X_train_ang.min():.3f}, {X_train_ang.max():.3f}] radians") print(f" (0 = qubit stays at |0⟩, π = qubit flipped to |1⟩)")

What each part does
scaler.fit_transform(X_train_all)Fit the scaler on training data only learn the min/max from training. Then transform training. Never fit on test data that would leak test statistics into the model.
scaler.transform(X_test_all)Apply the SAME transformation (learned from training) to test data. This ensures test features are in the same scale as training features. Critical for any ML pipeline.
pca.explained_variance_ratio_How much variance each principal component captures. If PC1 captures 30%, that means one number summarises 30% of all pixel variation across all images. The remaining 70%+ is discarded.
MinMaxScaler(feature_range=(0, np.pi))Scale PCA features to [0, π] the valid range for Ry rotation angles. An angle of 0 leaves the qubit at |0⟩. An angle of π rotates it to |1⟩. Values in between create superposition states.
pickle.dump({'scaler':..., 'pca':...})Save all fitted transformers to disk. The deploy scripts load this file and use the SAME transformers so the hardware sees identical preprocessing to what was used in training. This was a bug in the original code.
✓ What to look for in the PCA distributions

Trouser (class 1) should show a distinctly different distribution from other classes in PC1 it has a unique elongated shape that survives compression. T-shirt, Shirt, and Coat will heavily overlap because they have similar silhouettes. This overlap is why the VQC cannot reliably separate them the classes are not separable in 4-dimensional PCA space.

Import all dependencies, define the six training hyperparameters (qubits, layers, sample sizes, epochs, batch size, learning rate), then load Fashion-MNIST and filter down to T-shirt vs Trouser with random subsampling to 200 training and 100 test examples.

nb03_train_binary.pystep 1 imports + load preprocessed data
import numpy as np import pennylane as qml from pennylane import numpy as pnp import tensorflow as tf import pickle from sklearn.metrics import accuracy_score, confusion_matrix import matplotlib.pyplot as plt import warnings, time warnings.filterwarnings('ignore') # ── CONFIG ── N_QUBITS = 4 N_LAYERS = 2 N_TRAIN = 200 # 100 per class N_TEST = 100 # 50 per class N_EPOCHS = 40 BATCH_SIZE = 16 LR = 0.015 # ── Load and preprocess data ── print("[1/4] Loading Fashion-MNIST...") (X_train_all, y_train_all), (X_test_all, y_test_all) = \ tf.keras.datasets.fashion_mnist.load_data() X_train_all = X_train_all.reshape(-1,784).astype(np.float32) / 255.0 X_test_all = X_test_all.reshape(-1,784).astype(np.float32) / 255.0 # Filter T-shirt (0) and Trouser (1) def filter_binary(X, y): mask = (y==0) | (y==1) return X[mask], np.where(y[mask]==0, 0, 1) X_tr_raw, y_tr_all = filter_binary(X_train_all, y_train_all) X_te_raw, y_te_all = filter_binary(X_test_all, y_test_all) np.random.seed(42) X_tr_raw = X_tr_raw[np.random.choice(len(y_tr_all), N_TRAIN, replace=False)] y_tr = y_tr_all[np.random.choice(len(y_tr_all), N_TRAIN, replace=False)] X_te_raw = X_te_raw[np.random.choice(len(y_te_all), N_TEST, replace=False)] y_te = y_te_all[np.random.choice(len(y_te_all), N_TEST, replace=False)] print(f" Train: {N_TRAIN} | Test: {N_TEST}") # ── Preprocess fit on train, save transformers ── print("[2/4] Preprocessing: MinMaxScaler → PCA(4) → angle scaling...") from sklearn.decomposition import PCA from sklearn.preprocessing import MinMaxScaler scaler = MinMaxScaler() X_tr_n = scaler.fit_transform(X_tr_raw) X_te_n = scaler.transform(X_te_raw) pca = PCA(n_components=N_QUBITS, random_state=42) X_tr_pca = pca.fit_transform(X_tr_n) X_te_pca = pca.transform(X_te_n) print(f" PCA variance: {pca.explained_variance_ratio_.sum()*100:.1f}%") ang = MinMaxScaler(feature_range=(0, np.pi)) X_tr_ang = pnp.array(ang.fit_transform(X_tr_pca), requires_grad=False) X_te_ang = np.array(ang.transform(X_te_pca)) # plain numpy for classify() # FIX: save transformers so deploy uses identical preprocessing with open('binary_preprocessors.pkl', 'wb') as f: pickle.dump({'scaler':scaler, 'pca':pca, 'ang':ang}, f) print(" Saved: binary_preprocessors.pkl")

Create the PennyLane device and define the full VQC architecture the same encoding layer, variational Ry+Rz+CNOT ring structure, and ⟨Z₀⟩ measurement that you built in NB01 of M06, but now wired up with the MSE cost function and an Adam optimiser ready for training.

nb03_train_binary.pystep 2 define VQC architecture
# ── Build VQC ── print("[3/4] Building + training VQC...") dev = qml.device("default.qubit", wires=N_QUBITS) @qml.qnode(dev, interface="autograd") def vqc(weights, x): for i in range(N_QUBITS): qml.RY(x[i], wires=i) for l in range(N_LAYERS): for i in range(N_QUBITS): qml.RY(weights[l,i,0], wires=i) qml.RZ(weights[l,i,1], wires=i) for i in range(N_QUBITS-1): qml.CNOT(wires=[i, i+1]) qml.CNOT(wires=[N_QUBITS-1, 0]) return qml.expval(qml.PauliZ(0)) # +1 → T-shirt, −1 → Trouser def cost(W, X, y): preds = pnp.array([vqc(W, x) for x in X]) targets = pnp.where(y==0, 1.0, -1.0) return pnp.mean((preds - targets)**2) def classify(W, X): # FIX: convert to pnp array once, outside the loop X_pnp = pnp.array(X, requires_grad=False) preds = np.array([float(vqc(W, x)) for x in X_pnp]) return np.where(preds >= 0, 0, 1)

Run the 40-epoch Adam training loop each step draws a random mini-batch, computes gradients via the parameter shift rule, and updates the 16 weights; every 5 epochs we log train and test accuracy so you can watch the circuit learn to separate T-shirts from Trousers.

nb03_train_binary.pystep 3 training loop
np.random.seed(42) W = pnp.array(np.random.uniform(-0.01, 0.01, (N_LAYERS, N_QUBITS, 2)), requires_grad=True) opt = qml.AdamOptimizer(stepsize=LR) losses, tr_accs, te_accs, epochs_log = [], [], [], [] t0 = time.time() for ep in range(N_EPOCHS): idx = np.random.choice(N_TRAIN, BATCH_SIZE, replace=False) W, loss = opt.step_and_cost( lambda w: cost(w, X_tr_ang[idx], y_tr[idx]), W) if (ep+1) % 5 == 0: ta = accuracy_score(y_tr, classify(W, X_tr_ang)) tea = accuracy_score(y_te, classify(W, X_te_ang)) losses.append(float(loss)) tr_accs.append(ta); te_accs.append(tea) epochs_log.append(ep+1) print(f" Epoch {ep+1:3d}/{N_EPOCHS} | loss={float(loss):.4f} | train={ta:.3f} | test={tea:.3f}") total_time = time.time() - t0 # ── Final evaluation ── print("\n[4/4] Final evaluation...") final_preds = classify(W, X_te_ang) final_acc = accuracy_score(y_te, final_preds) cm = confusion_matrix(y_te, final_preds) print(f"\n{'='*40}") print(f" Binary VQC Simulator Results") print(f"{'='*40}") print(f" Test accuracy : {final_acc:.1%}") print(f" Random baseline: 50.0%") print(f" Training time : {total_time:.1f}s") print(f" Confusion Matrix:") print(f" Pred T-shirt Pred Trouser") print(f" True T-shirt {cm[0,0]:4d} {cm[0,1]:4d}") print(f" True Trouser {cm[1,0]:4d} {cm[1,1]:4d}") # ── Plot training curves ── fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(10, 4)) ax1.plot(epochs_log, losses, 'o-', color='#C0392B', linewidth=2) ax1.set_xlabel('Epoch'); ax1.set_ylabel('MSE Loss') ax1.set_title('Training Loss') ax2.plot(epochs_log, tr_accs, 'o-', color='#1A4FDB', linewidth=2, label='Train') ax2.plot(epochs_log, te_accs, 's--', color='#1A7A4A', linewidth=2, label='Test') ax2.axhline(0.5, color='gray', linestyle=':', alpha=0.5, label='Random') ax2.set_xlabel('Epoch'); ax2.set_ylabel('Accuracy') ax2.set_title('Accuracy'); ax2.set_ylim(0, 1.05); ax2.legend() fig.suptitle('Binary VQC Training Fashion-MNIST T-shirt vs Trouser', fontweight='bold') plt.tight_layout() plt.savefig('nb03_binary_training.png') plt.show() # ── Save weights + metadata ── np.save('binary_weights.npy', { 'W': np.array(W), 'losses': losses, 'tr_accs': tr_accs, 'te_accs': te_accs, 'final_acc': final_acc, 'cm': cm, 'n_qubits': N_QUBITS, 'n_layers': N_LAYERS }) print("\nSaved: binary_weights.npy + binary_preprocessors.pkl") print("→ Ready for NB 04: hardware deployment to IBM QPU")

The 10-class architecture adds a linear classification head on top of the VQC outputs. The VQC measures all 4 qubits giving [⟨Z₀⟩, ⟨Z₁⟩, ⟨Z₂⟩, ⟨Z₃⟩]. A 10×4 linear layer maps these 4 values to 10 class scores. Cross-entropy loss trains both the VQC parameters and the linear head.

import numpy as np import pennylane as qml from pennylane import numpy as pnp import tensorflow as tf import pickle from sklearn.decomposition import PCA from sklearn.preprocessing import MinMaxScaler from sklearn.metrics import accuracy_score import matplotlib.pyplot as plt import warnings, time warnings.filterwarnings('ignore') N_QUBITS = 4 N_LAYERS = 2 N_CLASSES = 10 N_PER_CLASS_TR = 50 # 500 total balanced N_PER_CLASS_TE = 20 # 200 total balanced N_EPOCHS = 60 BATCH_SIZE = 20 # 2 per class per batch LR_VQC = 0.01 LR_LIN = 0.05 CLASS_NAMES = ['T-shirt','Trouser','Pullover','Dress','Coat', 'Sandal','Shirt','Sneaker','Bag','Ankle boot'] # ── Load data ── print("[1/4] Loading Fashion-MNIST (balanced)...") (X_train_all,y_train_all),(X_test_all,y_test_all) = \ tf.keras.datasets.fashion_mnist.load_data() X_train_all = X_train_all.reshape(-1,784).astype(np.float32)/255.0 X_test_all = X_test_all.reshape(-1,784).astype(np.float32)/255.0 def balanced_sample(X, y, n): X_out, y_out = [], [] for c in range(N_CLASSES): idx = np.random.choice(np.where(y==c)[0], n, replace=False) X_out.append(X[idx]); y_out.append(np.full(n, c)) p = np.random.permutation(n*N_CLASSES) return np.vstack(X_out)[p], np.concatenate(y_out)[p] np.random.seed(42) X_tr_raw, y_tr = balanced_sample(X_train_all, y_train_all, N_PER_CLASS_TR) X_te_raw, y_te = balanced_sample(X_test_all, y_test_all, N_PER_CLASS_TE) print(f" Train: {len(y_tr)} | Test: {len(y_te)}") # ── Preprocess and save transformers ── print("[2/4] Preprocessing...") scaler = MinMaxScaler() X_tr_n = scaler.fit_transform(X_tr_raw) X_te_n = scaler.transform(X_te_raw) pca = PCA(n_components=N_QUBITS, random_state=42) X_tr_pca = pca.fit_transform(X_tr_n) X_te_pca = pca.transform(X_te_n) print(f" PCA variance: {pca.explained_variance_ratio_.sum()*100:.1f}%") ang = MinMaxScaler(feature_range=(0, np.pi)) X_tr_ang = pnp.array(ang.fit_transform(X_tr_pca), requires_grad=False) X_te_ang = np.array(ang.transform(X_te_pca)) with open('tenclass_preprocessors.pkl', 'wb') as f: pickle.dump({'scaler':scaler,'pca':pca,'ang':ang}, f) print(" Saved: tenclass_preprocessors.pkl") # ── VQC + linear head ── print("[3/4] Training VQC + linear head...") dev = qml.device("default.qubit", wires=N_QUBITS) @qml.qnode(dev, interface="autograd") def vqc(weights, x): for i in range(N_QUBITS): qml.RY(x[i], wires=i) for l in range(N_LAYERS): for i in range(N_QUBITS): qml.RY(weights[l,i,0], wires=i) qml.RZ(weights[l,i,1], wires=i) for i in range(N_QUBITS-1): qml.CNOT(wires=[i,i+1]) qml.CNOT(wires=[N_QUBITS-1,0]) return [qml.expval(qml.PauliZ(i)) for i in range(N_QUBITS)] def forward(vqc_w, lin_w, lin_b, X): scores = [] for x in X: q = pnp.array(vqc(vqc_w, x)) scores.append(pnp.dot(lin_w, q) + lin_b) return pnp.array(scores) def cost_fn(vqc_w, lin_w, lin_b, X, y): s = forward(vqc_w, lin_w, lin_b, X) exp = pnp.exp(s - pnp.max(s, axis=1, keepdims=True)) sm = exp / pnp.sum(exp, axis=1, keepdims=True) return -pnp.mean(pnp.log(sm[pnp.arange(len(y)), y] + 1e-9)) def classify(vqc_w, lin_w, lin_b, X): return pnp.argmax(forward(vqc_w, lin_w, lin_b, pnp.array(X, requires_grad=False)), axis=1) np.random.seed(42) vqc_w = pnp.array(np.random.uniform(-0.01,0.01,(N_LAYERS,N_QUBITS,2)), requires_grad=True) lin_w = pnp.array(np.random.randn(N_CLASSES,N_QUBITS)*0.1, requires_grad=True) lin_b = pnp.array(np.zeros(N_CLASSES), requires_grad=True) opt_vqc = qml.AdamOptimizer(stepsize=LR_VQC) opt_lin = qml.AdamOptimizer(stepsize=LR_LIN) losses, tr_accs, te_accs, ep_log = [], [], [], [] t0 = time.time() for ep in range(N_EPOCHS): # Balanced minibatch: 2 per class idx = np.concatenate([np.random.choice( np.where(y_tr==c)[0], BATCH_SIZE//N_CLASSES, replace=False) for c in range(N_CLASSES)]) Xb, yb = X_tr_ang[idx], y_tr[idx] # FIX: single joint update one cost_fn call per epoch (vqc_w, lin_w, lin_b), loss = opt_vqc.step_and_cost( lambda vw, lw, lb: cost_fn(vw, lw, lb, Xb, yb), vqc_w, lin_w, lin_b) # Extra linear head update (linear learns faster than VQC) (lin_w, lin_b), _ = opt_lin.step_and_cost( lambda lw, lb: cost_fn(vqc_w, lw, lb, Xb, yb), lin_w, lin_b) if (ep+1) % 5 == 0: ta = accuracy_score(y_tr, classify(vqc_w,lin_w,lin_b,X_tr_ang)) tea = accuracy_score(y_te, classify(vqc_w,lin_w,lin_b,X_te_ang)) losses.append(float(loss)); tr_accs.append(ta) te_accs.append(tea); ep_log.append(ep+1) print(f" Epoch {ep+1:3d}/{N_EPOCHS} | loss={float(loss):.4f} | train={ta:.3f} | test={tea:.3f}") total_time = time.time()-t0 # ── Final results + per-class breakdown ── print("\n[4/4] Final evaluation...") final_preds = np.array(classify(vqc_w,lin_w,lin_b,X_te_ang)) final_acc = accuracy_score(y_te, final_preds) per_class = {} for c in range(N_CLASSES): m = y_te==c per_class[CLASS_NAMES[c]] = accuracy_score(y_te[m], final_preds[m]) if m.sum()>0 else 0 print(f"\n{'='*45}") print(f" 10-Class VQC Simulator Results") print(f"{'='*45}") print(f" Test accuracy : {final_acc:.1%}") print(f" Random baseline: 10.0%") print(f" Training time : {total_time:.0f}s") print(f"\n Per-class accuracy:") for name, acc in per_class.items(): print(f" {name:12s} {acc:.2f} {'█'*int(acc*20)}") print(f"\n Paper's best on real hardware: ~40%") # ── Save ── np.save('tenclass_weights.npy', { 'vqc_w':np.array(vqc_w), 'lin_w':np.array(lin_w), 'lin_b':np.array(lin_b), 'losses':losses, 'tr_accs':tr_accs, 'te_accs':te_accs, 'final_acc':final_acc, 'per_class':per_class }) print("\nSaved: tenclass_weights.npy + tenclass_preprocessors.pkl") print("→ Ready for NB 06: hardware deployment to IBM Kingston")
Key fixes and design decisions
single step_and_cost callFIX: Original ran cost_fn twice per epoch once for VQC update, once for linear. This doubles runtime with minimal benefit since both share the same computation graph. Now one joint VQC update + one separate linear update.
LR_VQC=0.01, LR_LIN=0.05Separate learning rates. The linear head has no quantum circuit to evaluate its gradients are pure matrix operations and can take larger steps. VQC parameters need smaller steps because the loss landscape is more complex.
balanced minibatchPick exactly 2 samples per class per batch (20 samples / 10 classes). Without balancing the model can achieve 40%+ by learning only the easy classes (Trouser, Bag) and predicting them for everything.
lin_w shape: (10, 4)Linear layer maps 4 qubit expectation values to 10 class scores. Trained with cross-entropy + softmax standard multiclass classification. This is a hybrid quantum-classical architecture: quantum feature extraction + classical classification head.
tenclass_preprocessors.pkl savedFIX: Same issue as binary deploy. Save training-fitted scaler/PCA/ang transformers so NB 06 uses identical preprocessing to training.
✓ Expected results after 60 epochs

Overall accuracy: ~35-45%. Per-class: Trouser ~80-100% (distinctive shape in PCA), Bag ~60-80%, Dress ~50-70%, others lower. T-shirt, Shirt, Coat often 0-20% these are nearly indistinguishable in 4 PCA components. Training takes 60-90 minutes on CPU start it and come back.

What the analysis tells you
Binary at 45% below random (50%)The VQC learned to associate Trouser with the positive ⟨Z⟩ direction. But hardware noise pushed many T-shirt ⟨Z⟩ values negative predicting Trouser for everything. With 10/10 Trouser correct and only 0/10 T-shirt correct, accuracy = 50%... then noise pushes a few Trouser predictions wrong = 45%.
10-class at 30% above random (10%)The multi-qubit measurement + linear head is more robust. Even with noisy ⟨Zᵢ⟩ values, the linear layer finds the least-noisy decision boundary. Trouser still 100%, Dress/Bag at 67% the strong PCA signals survived the noise.
Depth 14 → 61 after transpileYour 4-qubit circuit must run on a 156-qubit chip with limited qubit connectivity. The transpiler adds SWAP gates to move qubit states to connected neighbours. Each SWAP = 3 CNOTs. 4× depth increase = 4× more accumulated gate noise.
85-90% sim-HW agreementHigh agreement means the circuits ran correctly the hardware understood what the simulator intended. The accuracy drop came from noise corrupting individual measurement outcomes, not from the circuit structure being wrong.

The pattern is not random. Green classes (Trouser, Dress, Bag) have distinctive shapes that survive PCA compression to 4 components. Red classes (T-shirt, Shirt, Coat, Pullover) overlap heavily in PCA space even the simulator only partially separates them. Hardware noise then pushes borderline predictions across the decision boundary. The result is a near-perfect classifier for Trouser and a coin flip for everything else.

📌 Why depth increases so much

Your circuit uses 4 qubits in a ring topology. IBM hardware has a fixed qubit connectivity graph not all pairs can directly CNOT. The transpiler adds SWAP gates to route qubit states to connected neighbours. Each SWAP costs 3 CNOTs. A circuit that needed 14 operations becomes 61 4× more gate noise.

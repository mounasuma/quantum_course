NB 01 · Classical only
The classical bit your baseline

1
Simulate a classical bit

A classical bit is just a variable that can be 0 or 1. Let's write one and read it 1000 times.

nb01_classical_bit.py python nb01_classical_bit.py
import matplotlib.pyplot as plt # A classical bit just a variable classical_bit = 0 # could also be 1 # "Read" it 1000 times simulating 1000 measurements results = [] for _ in range(1000): results.append(classical_bit) # Count how many 0s and 1s we got count_0 = results.count(0) count_1 = results.count(1) print(f"Got 0: {count_0} times") print(f"Got 1: {count_1} times") # Plot as a probability bar chart fig, ax = plt.subplots(figsize=(5, 4)) ax.bar(['|0⟩', '|1⟩'], [count_0/1000, count_1/1000], color=['#1A4FDB', '#6B3FA0'], alpha=0.7, width=0.5) ax.set_ylim(0, 1.1) ax.set_ylabel('Probability') ax.set_title('Classical bit = 0 1000 measurements') ax.axhline(0.5, color='gray', linestyle='--', alpha=0.4, label='50% line') plt.tight_layout() plt.savefig('nb01_output.png') plt.show()
What each part does
classical_bit = 0 Create a bit with value 0. This represents the physical bit in your computer a capacitor that is uncharged.
for _ in range(1000): Simulate reading the bit 1000 times. In a real computer, this would be 1000 memory reads.
results.append(classical_bit) Record the result of each read. Since the bit never changes, we always get 0.
ax.bar([...], [...]) Plot the probability of getting 0 vs 1 as a bar chart. This is the format we will use for all results in this course.
Expected output probability graph
Classical bit = 0 1000 measurements
|0⟩
value = 0
|1⟩
value = 1
0% 50% 100%
What you see: One bar at 100%, the other at 0%. The bit gave the same answer all 1000 times. This is what "deterministic" means same input, same output, every time. Remember this graph. In NB 03, it will look completely different.

The key property here: reading a classical bit does not change it. You can read it a million times and it stays 0. This seems obvious but a qubit behaves completely differently when measured. Keep this in mind.

1
Create a qubit and measure it

A qubit starts in the |0⟩ state by default the quantum equivalent of a bit set to 0. Let's measure it 1000 times and see what happens.

nb02_first_qubit.py python nb02_first_qubit.py
from qiskit import QuantumCircuit, transpile from qiskit_aer import AerSimulator import matplotlib.pyplot as plt # Step 1: Create a quantum circuit # QuantumCircuit(n_qubits, n_classical_bits) qc = QuantumCircuit(1, 1) # Step 2: Measure the qubit into the classical bit # qc.measure(qubit_index, classical_bit_index) qc.measure(0, 0) # Step 3: Draw the circuit so we can see what we built print("Circuit:") print(qc.draw()) # Step 4: Run on the Aer simulator, 1000 times sim = AerSimulator() job = sim.run(transpile(qc, sim), shots=1000) counts = job.result().get_counts() print("\nRaw counts:", counts) # Step 5: Convert to probabilities and plot prob_0 = counts.get('0', 0) / 1000 prob_1 = counts.get('1', 0) / 1000 fig, ax = plt.subplots(figsize=(5, 4)) ax.bar(['|0⟩', '|1⟩'], [prob_0, prob_1], color=['#1A4FDB', '#6B3FA0'], alpha=0.7, width=0.5) ax.set_ylim(0, 1.1) ax.set_ylabel('Probability') ax.set_title('Qubit in |0⟩ 1000 measurements') plt.tight_layout() plt.savefig('nb02_output.png') plt.show()
What each part does
QuantumCircuit(1, 1) Create a circuit with 1 qubit and 1 classical bit. The qubit holds the quantum state. The classical bit stores the measurement result.
qc.measure(0, 0) Measure qubit 0 and store the result in classical bit 0. This collapses the quantum state to 0 or 1.
AerSimulator() A classical simulation of quantum hardware running on your computer. Fast, noiseless, free. We use this for learning real hardware later.
transpile(qc, sim) Converts the circuit into instructions the simulator understands. Always required before running.
shots=1000 Run the circuit 1000 times. Each run is one measurement. We do many runs to build a probability distribution.
Expected output probability graph
Qubit in |0⟩ 1000 measurements
|0⟩
measured 0
|1⟩
measured 1
0%50%100%
Identical to the classical bit. A qubit in |0⟩ gives 0 every single time, just like a classical bit set to 0. This is not surprising the quantum behaviour only appears when we put the qubit in superposition. That's the next notebook.
NB 03 · Quantum
Superposition the Hadamard gate
One gate changes everything. The Hadamard gate puts a qubit into superposition and the probability graph transforms completely. This is where quantum starts to feel different.
⏱ ~20 minutes ⚛️ First superposition
By the end of this notebook
  • You understand what the Hadamard gate does mathematically
  • You have created a qubit in superposition and measured it
  • You can explain why the output is random not because of uncertainty but because of physics
  • You have seen the probability graph split into two equal bars
1
What does the Hadamard gate actually do?

The Hadamard gate (written as H) transforms |0⟩ into an equal superposition of |0⟩ and |1⟩. Mathematically:

H|0⟩ = (|0⟩ + |1⟩) / √2
This means: 50% probability of measuring 0, 50% probability of measuring 1

The √2 in the denominator is a normalisation factor it ensures the probabilities sum to 1 (since (1/√2)² + (1/√2)² = 0.5 + 0.5 = 1). You do not need to memorise this just understand that H creates a perfect 50/50 superposition.

2
Apply the Hadamard gate and measure
nb03_hadamard.py python nb03_hadamard.py
from qiskit import QuantumCircuit, transpile from qiskit_aer import AerSimulator import matplotlib.pyplot as plt qc = QuantumCircuit(1, 1) # Apply the Hadamard gate to qubit 0 # This is the ONLY difference from NB 02 one line changes everything qc.h(0) # Measure collapse the superposition qc.measure(0, 0) print("Circuit:") print(qc.draw()) sim = AerSimulator() counts = sim.run( transpile(qc, sim), shots=1000 ).result().get_counts() print("\nRaw counts:", counts) prob_0 = counts.get('0', 0) / 1000 prob_1 = counts.get('1', 0) / 1000 fig, ax = plt.subplots(figsize=(5, 4)) bars = ax.bar(['|0⟩', '|1⟩'], [prob_0, prob_1], color=['#1A4FDB', '#6B3FA0'], alpha=0.7, width=0.5) ax.set_ylim(0, 1.1) ax.set_ylabel('Probability') ax.set_title('H|0⟩ Superposition 1000 measurements') ax.axhline(0.5, color='red', linestyle='--', alpha=0.4, label='50% expected') ax.legend() plt.tight_layout() plt.savefig('nb03_output.png') plt.show()

Two roughly equal bars. Adding one gate qc.h(0) completely changed the probability distribution. The qubit is now in superposition: genuinely both 0 and 1 simultaneously. Every run of your code will give slightly different numbers but always near 50/50. This randomness is not a bug. It is physics.

NB 04 · Quantum
1000 measurements seeing probability in action
Run the same circuit at different shot counts and watch how the probability distribution converges. The more measurements you take, the closer you get to the true probability.
⏱ ~20 minutes 📊 Probability focus
By the end of this notebook
  • You understand why we need many shots, not just one measurement
  • You have plotted probability distributions at 10, 100, and 1000 shots
  • You can see how more shots → more accurate probability estimate
1
Why one measurement is not enough

When you measure a qubit in superposition, you get 0 or 1 one answer. That single answer tells you nothing about the probability distribution. You need to run the circuit many times to build a picture of the true probabilities.

This is like asking: "Is this coin fair?" You cannot tell from one flip. You need hundreds of flips to estimate the true probability.

nb04_shots.py python nb04_shots.py
from qiskit import QuantumCircuit, transpile from qiskit_aer import AerSimulator import matplotlib.pyplot as plt def measure_superposition(shots): """Run H|0⟩ circuit and return probability of 0 and 1.""" qc = QuantumCircuit(1, 1) qc.h(0) qc.measure(0, 0) sim = AerSimulator() counts = sim.run( transpile(qc, sim), shots=shots ).result().get_counts() return ( counts.get('0', 0) / shots, counts.get('1', 0) / shots ) # Run at 3 different shot counts shot_counts = [10, 100, 1000] results = {s: measure_superposition(s) for s in shot_counts} # Plot side by side one graph per shot count fig, axes = plt.subplots(1, 3, figsize=(12, 4)) for ax, shots in zip(axes, shot_counts): p0, p1 = results[shots] ax.bar(['|0⟩', '|1⟩'], [p0, p1], color=['#1A4FDB', '#6B3FA0'], alpha=0.7, width=0.5) ax.set_ylim(0, 1.1) ax.axhline(0.5, color='red', linestyle='--', alpha=0.4) ax.set_title(f'{shots} shots\np(0)={p0:.2f}, p(1)={p1:.2f}') ax.set_ylabel('Probability' if shots == 10 else '') plt.suptitle('H|0⟩ More shots → closer to true 50/50') plt.tight_layout() plt.savefig('nb04_output.png') plt.show()
Expected output three graphs side by side
How results converge as shots increase
10 shots
p(0)=0.70, p(1)=0.30 ← noisy
100 shots
p(0)=0.54, p(1)=0.46 ← getting closer
1000 shots
p(0)=0.51, p(1)=0.49 ← near perfect
The pattern: With only 10 shots, the result looks wildly uneven 70/30 instead of 50/50. With 100 shots it gets closer. With 1000 shots it converges near the true 50/50. This is why quantum computing uses many shots a single measurement gives you almost no useful information about the probability distribution.

from qiskit import QuantumCircuit, transpile from qiskit_aer import AerSimulator import matplotlib.pyplot as plt import numpy as np sim = AerSimulator() SHOTS = 1000 # ── Classical bit = 0 (no quantum, pure Python) ── classical_results = [0] * SHOTS classical_prob = [classical_results.count(0)/SHOTS, classical_results.count(1)/SHOTS] # ── Qubit in |0⟩ (no gate classical-like) ── qc_zero = QuantumCircuit(1, 1) qc_zero.measure(0, 0) c_zero = sim.run(transpile(qc_zero, sim), shots=SHOTS).result().get_counts() zero_prob = [c_zero.get('0',0)/SHOTS, c_zero.get('1',0)/SHOTS] # ── Qubit in superposition H|0⟩ ── qc_super = QuantumCircuit(1, 1) qc_super.h(0) qc_super.measure(0, 0) c_super = sim.run(transpile(qc_super,sim),shots=SHOTS).result().get_counts() super_prob = [c_super.get('0',0)/SHOTS, c_super.get('1',0)/SHOTS] # ── Plot all three side by side ── fig, axes = plt.subplots(1, 3, figsize=(12, 4)) labels = ['|0⟩', '|1⟩'] colors = ['#1A4FDB', '#6B3FA0'] for ax, probs, title, subtitle in [ (axes[0], classical_prob, 'Classical bit = 0', 'Deterministic'), (axes[1], zero_prob, 'Qubit in |0⟩', 'No superposition'), (axes[2], super_prob, 'Qubit: H|0⟩', 'Superposition!'), ]: bars = ax.bar(labels, probs, color=colors, alpha=0.75, width=0.45) ax.set_ylim(0, 1.15) ax.axhline(0.5, color='#ccc', linestyle='--', linewidth=0.8) ax.set_title(f'{title}\n({subtitle})', fontsize=11) ax.set_ylabel('Probability') for bar, p in zip(bars, probs): if p > 0.02: ax.text(bar.get_x() + bar.get_width()/2, bar.get_height() + 0.02, f'{p:.1%}', ha='center', fontsize=10) plt.suptitle('Classical bit vs qubit 1000 measurements each', fontsize=13, fontweight='bold') plt.tight_layout() plt.savefig('nb05_comparison.png') plt.show()
